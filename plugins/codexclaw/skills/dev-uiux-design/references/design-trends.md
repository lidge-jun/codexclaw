# Dated Design Trends

This skill uses three layers, each with one job:

1. **Isms** — `design-isms.md` owns visual and aesthetic direction.
2. **Compositional patterns** — `compositional-patterns.md` owns reusable hero, layout, navigation, motion, continuation, and content-structure families, including their canonical frequency, maturity, gates, and examples.
3. **Dated trends** — this file owns time-stamped axis prevalence and implementation-sensitive signals that may change on a later crawl.

Do not promote a dated signal into an ism or duplicate a compositional-pattern count here. Use `design-award-sources.md` for source selection and domain/type retrieval.

## Promotion pipeline

- **Signature → Emerging:** require independently coded evidence from **3+ sites across 2+ galleries**, a reusable definition, and named accessibility/performance risks. A gallery label alone is not evidence.
- **Emerging → Core:** require persistence across later crawls and source mixes, usefulness independent of a transient rendering technology, and stable gates/fallbacks. Clearing one crawl's numeric threshold is necessary but not sufficient.
- **Core retention or demotion:** if later evidence no longer persists or the apparent family collapses into a technology-specific signature, retain the dated record and demote current guidance rather than rewriting history.

Pattern-family promotion is applied in `compositional-patterns.md`; this file records the dated evidence and proposed transition only.

## Re-crawl protocol

1. **Freeze the source list:** record crawl date, corpus size, gallery names, per-gallery sample counts, and selection bias notes in `design-award-sources.md`-compatible form.
2. **Recode every site per axis:** independently code hero composition, motion approach, navigation pattern, typography stance, color mode, and layout structure. Add explicit technique fields for every signal whose current count says `not separately coded`.
3. **Aggregate without flattening:** count sites and distinct galleries per axis value and technique; preserve overlap and do not force an axis to sum to corpus size.
4. **Re-check thresholds:** test each candidate against 3+ sites / 2+ galleries, then evaluate persistence, source concentration, accessibility/performance risk, and technology dependence.
5. **Apply maturity transitions:** record Signature→Emerging, Emerging→Core, retention, or demotion with old and new dates/counts. Update this dated snapshot or append a newly dated snapshot; never silently replace prior evidence.
6. **Synchronize authorities:** pattern-level transitions update `compositional-patterns.md`; aesthetic implications update `design-isms.md` only when they are genuinely visual lineages; this file retains the dated technique evidence and cross-links.
