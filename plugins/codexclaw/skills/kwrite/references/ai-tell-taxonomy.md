# AI-Tell Taxonomy — 범용 한국어 윤문판

im-not-ai(epoko77-ai/im-not-ai) 계열 분류를 범용 윤문용으로 증류한 것.
예시 수정은 REGISTER-PRESERVING: 원문이 문어체면 문어체 안에서 고친다.
구어체 예시는 참고일 뿐, 문체 이식의 근거가 아니다.

## 심각도

- **S1**: 즉시 수정 — 읽는 사람이 바로 "AI가 썼네"라고 느끼는 패턴.
- **S2**: 수정 권장 — 누적되면 AI투로 읽힘.
- **S3**: 상황 판단 — 단독으로는 무해, 몰리면 문제.

## CAT-1: 번역투 (S1)

영어 문장 구조를 한국어에 그대로 옮긴 흔적.

| # | 패턴 | 예시 (전) | 문어체 수정 | 구어체 수정 |
|---|---|---|---|---|
| 1.1 | "~에 대해" | 속도에 대해 논의하면 | 속도를 논의하면 | 속도 얘기하면 |
| 1.2 | "~를 통해" | 이를 통해 알 수 있다 | 여기서 알 수 있다 | 그래서 ~인 거야 |
| 1.3 | "~함으로써" | 캐시를 사용함으로써 해결한다 | 캐시를 사용해 해결한다 | 캐시 쓰면 해결됨 |
| 1.4 | "~것이 중요하다" | 확인하는 것이 중요하다 | 반드시 확인해야 한다 | 확인해야 돼 |
| 1.5 | "~의 경우" | 사용자의 경우 | 사용자는 | 사용자는 |
| 1.6 | "~라는 점에서" | 빠르다는 점에서 유리하다 | 빨라서 유리하다 | 빨라서 좋음 |
| 1.7 | 관계절 좌향수식 | 최근 출시된 새로운 모델인 Sol은 | Sol이라는 신모델이 나왔는데, 이 모델은 | Sol 새로 나왔는데 |

## CAT-2: 기계적 병렬 (S2)

정보를 균등한 틀에 나열하는 습관. 자연스러운 글은 강약이 있다.

| # | 패턴 | 처방 |
|---|---|---|
| 2.1 | "첫째/둘째/셋째" | 순서 매김 없이 중요한 것부터 자연 전개 |
| 2.2 | "A, B, 그리고 C" | 항목마다 무게를 다르게: 하나는 문장으로, 나머지는 붙여서 |
| 2.3 | "~하며, ~하며, ~한다" | 문장 길이/구조 변주 |
| 2.4 | 불릿 남발 | 흐름이 있는 내용은 문장으로; 불릿은 진짜 목록에만 |

## CAT-3: AI 관용구 (S1)

AI가 즐겨 쓰지만 사람은 거의 쓰지 않는 표현.

| # | 패턴 | 문어체 대안 | 구어체 대안 |
|---|---|---|---|
| 3.1 | "시사하는 바가 크다" | 의미를 구체적으로 서술 | "이게 뭘 의미하냐면" |
| 3.2 | "주목할 만하다" | 왜 중요한지 한 문장으로 | "이게 진짜 중요한 건데" |
| 3.3 | "눈여겨볼 필요가 있다" | 삭제하고 본론 서술 | "이거 봐야 돼" |
| 3.4 | "~라고 할 수 있다" | 단정: "~이다" | 단정 |
| 3.5 | "~할 수 있을 것이다" | "~할 수 있다" 또는 단정 | "~될 거야" |
| 3.6 | "궤를 달리한다" | "수준이 다르다" + 근거 | "차원이 다름" |
| 3.7 | "이는 ~를 의미한다" | "즉 ~이다" (한 번만) | "그러니까 ~라는 거" |
| 3.8 | "결론적으로" | 삭제 — 마지막 문장이 곧 결론 | 삭제 |

| # | 패턴 | 처방 |
|---|---|---|

## 수정 원칙 (SKILL.md Prime directives의 실행판)

1. 탐지된 span만 수정한다.
2. 사실/수치/고유명사/인용/인과는 글자 단위로 보존한다.
3. 원문 대비 변경률 30%를 넘으면 멈추고 재검토한다.
4. 문체(격식/비격식, 문어/구어)는 원문 것을 유지한다.
