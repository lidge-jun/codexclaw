## Headline

The workflow ENGINE (PABCD FSM, attest gate, interview/contradiction) is at parity or
stricter than cli-jaw. The divergences are almost all driven by ONE architectural fact:
cli-jaw is a server runtime; codexclaw is a hooks + file-state Codex plugin. Server-shaped
operational surfaces (goal-write lifecycle, memory federation, task DB, dashboard, deploy)
are documented non-goals, not unfinished work. The single genuine defect is the loop⇄goal
arming break already tracked in `140_L14`.

## Detail by verdict

### Parity or stricter (engine)

- **PABCD FSM + attest (PARITY).** `VALID_TRANSITIONS` edge set and array order match
  cli-jaw exactly; the four gated edges, placeholder-`did` rejection, and the C→D
  `checkOutput` + nonzero `exitCode` requirement are equivalent. No gap found.
- **Interview / Minds (codexclaw STRICTER).** Same four dimensions (goal/constraint/
  success/ontology). cli-jaw enforces readiness only via prompt text and lets I→P proceed
  with a soft warning; codexclaw hard-gates I→P (`scanRounds>=1` + all dims max + zero
  contradictions + override-only) and adds the 5-Mind contradiction lenses, severity
  triage, and fail-closed reconstruction that cli-jaw has no equivalent for.
- **evidence / freeze (PARITY + EXTRA).** Same attest gate; codexclaw additionally
  persists `ledger.jsonl` + per-session interview JSONL and a freeze manifest (planHash),
  where cli-jaw checkpoints are in-memory only.

### Partial parity / structural difference

- **Loop continuation.** "1 work-phase = 1 PABCD cycle" holds on both sides. The arming
  layer differs: codexclaw arms in the Stop hook (phase-local, bounded by MAX_STOP_BLOCKS);
  cli-jaw arms at lifecycle exit via a goal heartbeat (goal-wide), plus a two-tap pause
  governor and delayed respawn timer with no codexclaw counterpart. The loop⇄goal arming
  break is the one real defect — tracked in `140_L14` (14.1/14.2).
- **Subagent / role / model.** Different axis: codexclaw has fixed roles
  explorer/reviewer/executor with `mode/model/promptOverride` persistence and a
  spawn-wrapper; cli-jaw has an employee DB + static employees + frontend/backend/data/docs
  taxonomy. Not 1:1.

### Intentional non-goals (architecture-forced)

- **goal lifecycle.** cli-jaw owns 11 write subcommands + a done evidence gate; codexclaw
  reads native `goals_1.sqlite` READ-ONLY (active/inactive/unreadable) and delegates goal
  creation to the main session by design.
- **memory / chat.** cli-jaw memory CRUD/search/context + cross-instance federation are
  out; chat-search was retired in L13/WP1. `cxc-search` is external/public-web only.
- **task / project / worklog.** task DB replaced by native `update_plan`; no project
  workspace manager; worklog absorbed into `ledger.jsonl` + `devlog/_plan/`.
- **architecture / deploy / dashboard.** cli-jaw server (3457), dashboard, Electron, Docker
  vs codexclaw hooks + file-state plugin + local GUI. Forced non-goal.
- **CLI surface.** Only `orchestrate` is an exact match; goal/memory/task/project/dashboard/
  employee/etc. are absent (native-delegated or non-goal).
