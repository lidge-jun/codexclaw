# L12.1 / 121 — Interview Ledger + PostToolUse Answer Capture (runtime impl)

Status: DONE (shipped + tested) · 2026-06-30 · mvp_hard loop L12.1 · class C3

> L12 (120) reconciled the skill surfaces and DEFERRED the interview runtime to
> "L13+". This sub-loop ships the first deferred piece: durable capture of the
> `request_user_input` question/answer rounds.

## Gap

`request_user_input` was prompted by `hook.ts` (UserPromptSubmit injects the
interview directive), but nothing recorded the question/options/answer once the
user replied. The scan-evidence ledger (state.ts `appendInterviewEvent`) only
proves a contradiction scan ran — it carried no Q/A history.

## Implementation

- New `pabcd-state/src/interview-ledger.ts`:
  - `question_asked` + `answer_recorded` events in the per-session
    `.codexclaw/interviews/<sessionId>.jsonl` (shared with scan events, keyed by
    the `event` discriminator).
  - event ids DERIVED from `(turnId, questionId, eventKind)` so a re-fired hook in
    the same turn does not double-record; a new turn re-asking is a fresh event.
  - `parseQuestions`/`parseAnswers` read the real `request_user_input` shape
    (`tool_input.questions[{id,header,question}]`,
    `tool_response.answers{<id>:{answers:[]}}`) — total, never throw.
- `handlePostToolUse` (hook.ts) + `parsePostToolUse` (parse.ts) + a
  `post-tool-use` dispatch branch in cli.ts (inside the fail-safe try).
- New hook `post-tool-use-capturing-interview-answers.json` (matcher
  `^request_user_input$`), registered in `plugin.json`.

## Conflict check (PreToolUse goal-deny vs PostToolUse capture)

No conflict. The PreToolUse interview deny is a SEPARATE hook that fires BEFORE the
tool executes; when a goal is active it denies `request_user_input`, so the tool
never runs and PostToolUse never fires. PostToolUse capture therefore only records
rounds that were actually allowed (goal inactive / interactive interview). The
PostToolUse handler is a pure recorder — it has no decision/block output and always
returns "".

## Acceptance (proven by interview-ledger.test.ts + e2e cli.js)

- records a `question_asked` + `answer_recorded` per answered question;
- idempotent for the same (turn, question, kind); a new turn re-records;
- unanswered question records only `question_asked`;
- malformed payloads / missing session fail safe (no events, no throw);
- `handlePostToolUse` captures only `request_user_input` and returns "" (never blocks);
- e2e through the compiled `dist/cli.js hook post-tool-use` writes the session JSONL.
