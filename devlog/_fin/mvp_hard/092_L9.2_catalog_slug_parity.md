# L9.2 / 092 — Catalog Slug Parity (runtime impl)

Status: DONE (shipped + tested) · 2026-06-30 · mvp_hard loop L9.2 · class C2

> Closes L9.2 from the 090 slice map: the native catalog cache reader missed
> entries keyed by `slug`, which is how the LIVE Codex catalog stores natives.

## Gap

`readNativeCacheDefault` mapped each cache entry to its `id` only. But the live
Codex catalog keys every model by `slug` (bare like `gpt-5.5`, or routed
`provider/model`), not `id` — opencodex `src/codex-catalog.ts:152,183` filters and
reads on `slug`. So a real Codex `CODEX_MODELS_CACHE_PATH` file (entries shaped
`{ slug: "gpt-5.5", base_instructions: ... }`) yielded an empty read and silently
fell back to the static `NATIVE_OPENAI_MODELS` set, hiding the user's actual
installed natives.

## Fix

- `entryKey(m)` resolves a raw entry to a stable key: bare string, else `id`,
  else `slug`.
- `readNativeCacheDefault` now reads `id || slug`, keeps the same
  `NATIVE_OPENAI_MODELS` allowlist, and dedups first-seen so a slug+id duplicate
  for one model collapses.
- `buildCatalog` was already id-keyed and native-first; routed `provider/model`
  ocx slugs flow through as `source:"ocx"` entries and dedup against native.

## Acceptance (all proven by catalog.test.ts)

- Reads entries keyed by `slug` (live Codex shape); legacy/internal natives and
  routed slugs are filtered by the native allowlist.
- `id` and `slug` for the same model collapse to one allowlisted entry.
- Routed `provider/model` ocx slugs are selectable, deduped, native first.
- Existing id-keyed behavior, fallback set, and unsupported-ocx state preserved.
