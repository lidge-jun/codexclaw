## M1 — 질문 UX = ooo식 + cli-jaw식 (둘 다 채택)
모든 인터뷰 질문은 다음을 반드시 포함한다(cli-jaw 인터뷰 UX 차용):
1. **배경 설명 (왜 이걸 묻나)** — 아주 친절하게. 어떤 모순/가정/누락에서 이 질문이 나왔는지,
   무엇이 걸려 있는지(영향), 결정이 어디로 흘러가는지 먼저 서술.
2. **질문 설명 (아주 구체적으로)** — 각 선택지가 실제로 무엇을 의미하는지, trade-off가 무엇인지
   구체적으로. 추상적 라벨만 던지지 않는다.
3. **선택지** — 2~3개, 권장안 먼저. (cli-jaw structured elicitation 형식.)
- ooo식 = Socratic(한 겹 더 파기) + Contrarian(반대가 참이면?) 으로 모순을 먼저 노출한 뒤 질문화.

## M2 — 루프 = 질문 → 서브에이전트 → 질문 (반복)
- 메인이 질문 던지고 jun 답 수집 → 서브에이전트로 모순 재도출 → 새 모순을 다시 질문화. 계속 반복.
- jun 답과 결정은 매 라운드 devlog에 즉시 기록(080.1 레지스터 + 080 RESOLVED 섹션).

## M3 — 서브에이전트 재활용 + 원점(fresh-eyes) 프롬프트 강제
- **컨텍스트 보존**을 위해 같은 모순-도출 서브에이전트를 재활용(send_input)하되,
- 매 호출에 **원점에서 보게 하는 프롬프트를 강하게 주입**한다. 누적 편향/이전 결론 답습 방지:
  > "이전 라운드 결론을 신뢰하지 말고 회의적 외부자처럼 처음부터 재평가하라. 직전에 'OK'였던 것도
  >  다시 의심하라. 점수·합의를 빨리 닫기 위해 올리지 마라."
  (cli-jaw 인터뷰의 'Re-assess from scratch / Negativity bias' 규율 차용.)
- 즉 "context는 보존, 판단은 매번 원점". 재활용으로 토큰·맥락 이득 + 강한 리셋 프롬프트로 편향 제거.
