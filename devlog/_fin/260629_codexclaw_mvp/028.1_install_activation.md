# 028.1 — Install / Activation Flow

Status: TODO  ·  Phase 1  ·  (see 022.2, 027)

## Goal
Define what `codex plugin add codexclaw` + codexclaw's own enable step does, including the
documented feature-flag activation.

## Flow
1. `codex plugin marketplace add <repo>` + `codex plugin add codexclaw@personal`.
2. codexclaw enable (CLI command or SessionStart bootstrap):
   - back up `~/.codex/config.toml`.
   - `codex features enable multi_agent goals hooks default_mode_request_user_input` (only those
     not already true).
   - register codexclaw skills/hooks/agents (via plugin manifest).
3. Verify activation: `codex features list` shows the four enabled.
4. `codexclaw uninstall`: revert exactly the flags codexclaw set; remove plugin.

## Notes
- `default_mode_request_user_input` is "under development" but currently defaults true; treat as
  soft dependency (Q-IPABCD-2): if off/unavailable, Interview degrades to plain text Q&A.
- This is the ONLY config.toml write codexclaw performs (027).

## Verify
- Fresh install enables the four flags + creates backup.
- Uninstall restores prior flag state.
