## Goal (one slice)
Document and ship the B-opt2 contract: cxc role TOMLs are prompt sources, not auto-registered roles.

## Why now / dependencies
The role files exist from L5.1. The implementation decision that makes Phase 1 robust is that cxc
injects their developer instructions inline when spawning built-in Codex agents.

## Scope (decision-complete)
- Files added/edited:
  - `plugins/codexclaw/agents/README.md`
  - `plugins/codexclaw/agents/explorer.toml`
  - `plugins/codexclaw/agents/reviewer.toml`
  - `plugins/codexclaw/agents/executor.toml`
- Role to built-in mapping:
  - `explorer` -> Codex `agent_type: "explorer"`
  - `reviewer` -> Codex `agent_type: "explorer"`
  - `executor` -> Codex `agent_type: "worker"`
- Inline spawn contract:
  - `spawn_agent({ agent_type: "explorer", message: "TASK: <role instructions + task>" })`
  - `spawn_agent({ agent_type: "worker", message: "TASK: <role instructions + scoped task>" })`
- Rationale:
  - Codex plugin manifests expose `skills`, `hooks`, `mcpServers`, and `apps`.
  - Phase 1 does not assume a plugin manifest `agents` field.
  - TOMLs stay canonical so future config-layer registration can reuse them.
- Must-NOT-Have:
  - No claim that plugin install auto-registers these roles.
  - No live spawn proof in offline L7 gate.
  - No write permission for `explorer` or `reviewer`.
