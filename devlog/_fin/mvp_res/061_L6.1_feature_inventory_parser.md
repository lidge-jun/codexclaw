## Goal (one slice)
Define the exact Codex feature flags cxc owns and parse `codex features list` safely.

## Why now / dependencies
Activation can only be safe if cxc knows which flags it is allowed to change. This sub-pass prevents
substring collisions before activate/deactivate mutate anything.

- Parsing behavior:
  - reads the first field in the `codex features list` table as the feature name.
  - records only exact declared feature names.
  - treats unseen declared flags as disabled.
  - throws on nonzero runner exit.

  - No ownership of `plugin_hooks`.
  - No hardcoded parse of sibling rows.
