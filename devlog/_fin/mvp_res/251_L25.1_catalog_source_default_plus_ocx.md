# L25.1 (Decade 251) -- Catalog Source Native Plus ocx

Status: DONE
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 250_L25_model_catalog.md, 260629_codexclaw_mvp/033_model_catalog.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Define the selectable model catalog source as Codex's native catalog plus models
exposed by the optional external ocx provider bridge.

## Why now / dependencies
- Upstream: depends on L23 provider status and L24 schema destination for model
  IDs.
- Downstream: feeds L27 model selectors and L28 S7 verification.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/components/subagent-config/src/catalog.ts`
  - `plugins/codexclaw/components/subagent-config/src/mcp.ts`
  - `plugins/codexclaw/components/subagent-config/test/catalog.test.ts`
  - provider-bridge status reader only if L23 exposes catalog metadata there
- Exact behavior:
  - Always include Codex-native catalog entries.
  - When ocx is detected and exposes a catalog, append ocx-backed entries.
  - Deduplicate by stable model ID with native entries first.
  - Include metadata that labels native versus ocx-backed entries.
  - If ocx catalog interface is unavailable, report unsupported-catalog state.
- Must-NOT-Have:
  - No vendored ocx catalog files.
  - No arbitrary provider network fetch.
  - No selected-model writes; L24 owns persistence.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: design catalog entry shape, native entry metadata, ocx adapter contract,
  dedupe rules, and unsupported-state shape.
- A: audit angle = "does the catalog fabricate provider data?" Reviewer checks
  absent, present, duplicate, and unsupported fixtures.
- B: implement catalog builder and adapter boundary with fixture inputs.
- C: run node:test catalog fixtures and dump absent/present catalog JSON.
- D: done = S7 has a deterministic native-plus-ocx catalog source.

## Acceptance (1-3 testable criteria)
1. ocx absent returns Codex-native catalog entries.
2. ocx present with n fixture models returns native+n entries with native
   entries first.
3. Duplicate IDs do not remove native catalog entries.

## QA channel (node:test path / CLI stdout / tmux / data dump)
- node:test path: `plugins/codexclaw/components/subagent-config/test/catalog.test.ts`
- Data dump: catalog JSON for absent, present, duplicate, and unsupported cases.

not a single fallback model. Ground truth: opencodex

["gpt-5.5","gpt-5.4","gpt-5.4-mini","gpt-5.3-codex-spark"]`, read from the
Codex live catalog cache (`CODEX_MODELS_CACHE_PATH`) through the allowlist.
