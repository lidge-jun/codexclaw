# L1.3 (Decade 013) -- Session Storage and Ledger

Status: DONE
Cluster: 1 - Phase: 1 - Shorthand: cxc
Source-of-record: 016_session_scope_finding.md; 018_pass1_P_plan.md; 022.1_pabcd_state_files.md

## Goal (one slice)
Persist IPABCD state per Codex session and provide a shared append-only ledger file.

## Why now / dependencies
The session-scoped finding was a hard dependency for all Phase 1 hooks. L2 needs per-turn idempotency
inside a session file, while ledger append remains available for future phase-transition evidence.

## Scope (decision-complete)
- Files added/edited:
  - `plugins/codexclaw/components/pabcd-state/src/state.ts`
  - `plugins/codexclaw/components/pabcd-state/test/state.test.ts`
- Paths shipped:
  - `STATE_DIR = ".codexclaw"`
  - `SESSIONS_SUBDIR = "sessions"`
  - `LEDGER_FILE = "ledger.jsonl"`
  - session file: `<cwd>/.codexclaw/sessions/<sanitizeKey(sessionId)>.json`
  - ledger file: `<cwd>/.codexclaw/ledger.jsonl`
- Storage behavior:
  - `sanitizeKey` keeps only safe filename chars and falls back to `missing`.
  - `readState` returns `defaultState(sessionId)` on missing, corrupt, or invalid JSON.
  - `writeState` creates the sessions dir, writes a pid/timestamp temp file, then atomic renames.
  - failed write cleanup removes the temp file best-effort before rethrowing.
  - `appendLedger` writes one JSON object per line, including `sessionId`.
- Must-NOT-Have:
  - No per-cwd single state file.
  - No ledger write from passive Stop in Phase 1.
  - No silent mutation of unrelated session files.

- A corrupt session file never blocks cxc; `readState` returns default `I`.
- `appendLedger` creates `.codexclaw/ledger.jsonl` with session-tagged NDJSON lines.
