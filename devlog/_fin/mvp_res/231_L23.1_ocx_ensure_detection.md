# L23.1 (Decade 231) -- ocx Detection

Status: DONE
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 230_L23_provider_bridge.md, 260629_codexclaw_mvp/031_provider_bridge.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Detect optional opencodex (`ocx`) at SessionStart and read the external
subagent/model list only when a real ocx installation or service is available.

## Why now / dependencies
- Upstream: depends on L6 install/activation so the SessionStart hook can run
  and on L23 head scope for provider-bridge placement.
- Downstream: gives L25 and L27 a trustworthy provider status source for model
  catalog and 10100 link-bar decisions.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/components/provider-bridge/src/cli.ts`
  - `plugins/codexclaw/components/provider-bridge/test/cli.test.ts`
  - `plugins/codexclaw/hooks/session-start-ensuring-provider-bridge.json`
- Exact behavior:
  - Resolve `ocx` from PATH first.
  - If PATH lookup fails, check only the known opencodex service surface called
    out by the provider bridge plan.
  - When detected, read ocx's external subagent/model list and expose structured
    provider status for downstream consumers.
  - Treat opencodex as optional and external; never copy or vendor it.
- Must-NOT-Have:
  - No invented provider proxy if ocx is missing.
  - No direct mutation of Codex global config or auto-ensure behavior.
  - No dependence on the opencodex GUI source tree.

## IPABCD micro-cycle
- I: not interview-bearing; Q-P2-2 is the only open policy fork.
- P: split detection into PATH lookup, optional service probe, list read,
  and status emission.
- A: audit angle = "does detection prove ocx is external and real before
  reading model/subagent data?" Reviewer checks stubs for PATH-present and
  service-present.
- B: implement detection helpers, wire SessionStart CLI entry, add fixture ocx
  command tests, and keep status machine-readable.
- C: run node:test with a stub `ocx` that records list reads; run the hook
  command directly and capture stdout.
- D: done = S6 present path passes: detected ocx is read exactly once and
  reports multi-provider availability.

## Acceptance (1-3 testable criteria)
1. With a stub `ocx` on PATH, SessionStart detects ocx and reads fixture
   subagent/model data exactly once.
2. Provider status identifies ocx as external and detected without vendored
   files.
3. A failing ocx list read is reported as a read failure, not as ocx absent.
