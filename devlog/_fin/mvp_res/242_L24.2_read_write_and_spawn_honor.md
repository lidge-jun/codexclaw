# L24.2 (Decade 242) -- Read/Write API and Spawn Honor

Status: DONE
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 240_L24_subagent_config_store.md, 260629_codexclaw_mvp/032_subagent_config_store.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Implement config read/write APIs and make spawned explorer/reviewer/executor
subagents honor configured model mode and prompt override.

## Why now / dependencies
- Upstream: depends on L24.1 schema and Phase-1 role spawn boundaries.
- Downstream: unblocks L27 save/load behavior and L28 S8/S10 verification.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/components/subagent-config/src/store.ts`
  - `plugins/codexclaw/components/subagent-config/src/mcp.ts`
  - `plugins/codexclaw/components/subagent-config/test/store.test.ts`
  - `plugins/codexclaw/components/subagent-config/test/mcp.test.ts`
  - smallest existing Phase-1 subagent spawn integration point
- Exact behavior:
  - Writes use temp-file then rename for local atomicity.
  - `mode: default` ignores `model` and uses the main Codex model.
  - `mode: model` requires a selected catalog model.
  - `promptOverride` applies only to the role prompt segment owned here.
  - Spawn resolution reads current config at spawn time.
- Must-NOT-Have:
  - No prompt override replacing unrelated system or dev-skill instructions.
  - No silent fallback from malformed config to a wrong explicit model.
  - No direct React writes to the file outside this API.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: add write/update methods, MCP/tool surface for GUI, and spawn-time resolver.
- A: audit angle = "does GUI-persisted state match actual subagent spawn?"
  Reviewer checks model mode, prompt text, validation, and default behavior.
- B: implement writer, readback, resolver, API tests, and spawn integration.
- C: run node:test for store/MCP; run a data dump showing saved reviewer config
  and resolved spawn config.
- D: done = S8 and S10 have a working persistence-to-spawn path.

## Acceptance (1-3 testable criteria)
1. Updating reviewer to `mode: model` persists and reads back exactly.
2. A spawned reviewer receives the selected model while default-mode roles use
   the main model.
3. A prompt override is included in the role prompt segment and nowhere else.
