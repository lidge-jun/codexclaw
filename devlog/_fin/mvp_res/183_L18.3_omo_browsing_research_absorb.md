## Goal (one slice)
Absorb omo `ultimate-browsing` and `ultraresearch` into codexclaw search without
copying unsupported infrastructure. `ultimate-browsing` becomes a blocked-source
reader helper under Tier 2; `ultraresearch` becomes the opt-in Tier 3 orchestration
pattern for deep research.

## Why now / dependencies
- Upstream: L18.1 defines the three-tier ladder and L18.2 defines intent routing.
- J-10 requires codex-only backends, so omo skills must be translated into
  patterns and references rather than copied as-is.
- Downstream: L21 librarian/external explorer variant can point at this sub-pass
  when it needs long-form OSS/docs research.

## Scope (decision-complete)
Files to add/edit in implementation:
- `plugins/codexclaw/skills/search/SKILL.md`
- `plugins/codexclaw/skills/search/references/blocked-url-reader.md` (omo ultimate-browsing absorb)
- `plugins/codexclaw/skills/search/references/deep-research-swarm.md` (omo ultraresearch absorb;
  breadth/depth criteria for multi-query research)

Exact behavior:
- `ultimate-browsing` contributes tactics for blocked, JS-rendered, PDF, table,
  or otherwise hard-to-read sources. These tactics live in `blocked-url-reader.md`
  and are invoked only after a candidate URL exists.
- The blocked-reader helper is not a fourth tier. It is Tier 2 Browser Use /
  Computer Use guidance for reaching and validating a source.
- `ultraresearch` contributes research decomposition: parallel source classes,
  librarian/explorer assignments, source quality grading, contradiction capture,
  and synthesis with citations.
- Ultraresearch mode requires explicit main-agent decision because it consumes
  subagents and time. It is appropriate for broad comparative research, ecosystem
  surveys, or difficult public evidence gathering, not ordinary single-answer
  lookup.
- If Tier 3 is used, subagents must be scoped: one query family or source class
  per agent, source URLs returned, no edits, no hidden web providers.
- The search skill must say when to stop: sufficient primary source found, all
  candidate URLs dead, or the task needs user clarification.

Must-NOT-Have:
- No direct promise to run omo-specific web-ai, progrok, CloakBrowser, or hidden
  browser vendors.
- No durable background research claim; Codex subagents are waited in-turn unless
  a later scheduler feature owns durability.
- No external-source summary without source URL and opened-source verification.
