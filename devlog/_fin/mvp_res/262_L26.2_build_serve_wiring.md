# L26.2 (Decade 262) -- Build and Serve Wiring

Status: DONE
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 260_L26_gui_scaffold.md, 260629_codexclaw_mvp/034_gui_scaffold.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Wire `cxc gui` and GUI build/serve commands so the local codexclaw dashboard can
launch and print a reachable URL.

## Why now / dependencies
- Upstream: depends on L26.1 app scaffold and L6 binary alias conventions.
- Downstream: lets L27 be opened by users and lets L28 verify GUI build and
  launch behavior.

## Scope (decision-complete)
- Files to add/edit:
  - `cli/package.json`
  - CLI launcher wiring for `cxc gui`
  - `plugins/codexclaw/gui/package.json`
  - `plugins/codexclaw/gui/README.md`
  - build output wiring only if existing plugin packaging requires it
- Exact behavior:
  - `cxc gui` starts the local dashboard and prints the URL.
  - Serve path does not require ocx or opencodex dashboard.
  - Build path runs from `plugins/codexclaw/gui`.
  - GUI reads/writes only through codexclaw APIs.
  - opencodex GUI remains a layout/structure reference only; serving uses
    codexclaw-owned content and endpoints.
- Must-NOT-Have:
  - No permanent daemon in Phase 2.
  - No automatic launch of opencodex GUI.
  - No global Codex config mutation from GUI serving.

## IPABCD micro-cycle
- I: not interview-bearing; Q-P2-1 resolved opencodex as layout/structure
  reference only.
- P: add CLI subcommand, process lifecycle, URL stdout, and build script hooks.
- A: audit angle = "does launcher start codexclaw GUI only?" Reviewer checks
  command path, port assumptions, and shutdown behavior.
- B: implement launcher, npm scripts, README instructions, and smoke command.
- C: run GUI build; run `cxc gui` briefly and verify URL output plus clean
  shutdown.
- D: done = dashboard can be served locally through the cxc primary CLI.

## Acceptance (1-3 testable criteria)
1. `cxc gui` prints a local URL and serves the codexclaw app.
2. GUI build completes from `plugins/codexclaw/gui`.
3. Serving works with ocx absent and does not open `localhost:10100`.

## QA channel (node:test path / CLI stdout / tmux / data dump)
- CLI stdout: `cxc gui` startup URL and shutdown result.
- GUI build output from `plugins/codexclaw/gui`.
- Browser/console smoke check for served app shell.

## Commit unit (one atomic conventional commit)
`feat(gui): wire cxc gui launcher`

## Blocked-on (jun decision id, if any)
None. Q-P2-1 resolved: opencodex GUI is a layout/structure reference only; do
not copy, vendor, or import opencodex GUI files.

## Resolved (jun 2026-06-30)
Q-P2-1 resolved GUI reuse policy: codexclaw builds and serves its own GUI while
borrowing only layout patterns from opencodex. opencodex GUI is API-driven (for
example, `gui/src/pages/Subagents.tsx` fetches `/api/*`), so `cxc gui` must serve
codexclaw-owned content and codexclaw endpoints, not copied opencodex files.
