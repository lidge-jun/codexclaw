# L1.1 (Decade 011) -- State Schema

Status: DONE
Cluster: 1 - Phase: 1 - Shorthand: cxc
Source-of-record: 018_pass1_P_plan.md; 022.1_pabcd_state_files.md; STATUS.md

## Goal (one slice)
Define the durable IPABCD state object that every later cxc hook and guard can read safely.

## Why now / dependencies
This is the first atomic part of L1. L1.2 FSM predicates and L2 directive injection both depend on
the `Phase`, `State`, and `Flags` contracts exported here.

## Scope (decision-complete)
- Files added/edited:
  - `plugins/codexclaw/components/pabcd-state/src/state.ts`
  - `plugins/codexclaw/components/pabcd-state/test/state.test.ts`
- Public schema:
  - `Phase = "I" | "P" | "A" | "B" | "C" | "D"`
  - `PHASES = ["I", "P", "A", "B", "C", "D"]`
  - `Flags = { interview, auditPassed, checkPassed }`
  - `State = { phase, sessionId, slug, updatedAt, flags, supersededBy, injectedTurns,
    lastInjectedPhase, orchestrationActive }`
  - `LedgerEntry = { ts, sessionId, from, to, reason, evidence? }`
- Functions shipped:
  - `sanitizeKey(value)`
  - `defaultState(sessionId, slug = "")`
  - `readState(cwd, sessionId)`
  - `writeState(cwd, next)`
  - `appendLedger(cwd, entry)`
- Must-NOT-Have:
  - No global singleton.
  - No unknown-key passthrough from persisted JSON.
  - No runtime hook behavior in this sub-pass.

## IPABCD micro-cycle
- I (if interview-bearing): not interview-bearing; interview is represented only by `flags.interview`.
- P: put `PHASES` beside `Phase` in `state.ts` so FSM imports one-way and avoids a cycle.
- A: Pass 1 audit found the ORDER/Phase ownership risk and accepted strict schema ownership here.
- B: implemented TypeScript interfaces plus default construction and strict reconstruction.
- C: state tests cover defaults, invalid persisted phase, unknown-key dropping, and new injection fields.
- D: done = schema is the single exported source for phase values and state shape.

## Acceptance (1-3 testable criteria)
- `defaultState("s")` returns phase `I`, all flags false, and no active orchestration.
- Persisted unknown keys are dropped when `readState` reconstructs a state.
- Invalid `phase`, `lastInjectedPhase`, or non-boolean orchestration fields fall back safely.
