# L20.3 (Decade 203) -- cxc doctor/reset Self-Implementation

Status: DONE

## Goal (one slice)
Define the first plugin-owned `cxc` commands: `cxc doctor` for evidence-bound
health reporting and `cxc reset` for scoped codexclaw cleanup. These commands are
self-implemented because Codex cannot inspect codexclaw plugin policy, generated
state, and skill payload health on its own.

## Why now / dependencies
- Upstream: L6 creates the `cxc` binary alias and activation path.
- L20 owns cli-jaw command mapping; doctor/reset are the clear gaps not covered
  by Codex-native tools.
- Downstream: L21 folds lcx-doctor/test-support diagnostics into these commands,
  and install/activation checks can call `cxc doctor`.

## Scope (decision-complete)
Files to add/edit in implementation:
- `plugins/codexclaw/bin/codexclaw.mjs` or CLI entry equivalent
- `plugins/codexclaw/src/cli/doctor.*`
- `plugins/codexclaw/src/cli/reset.*`
- `plugins/codexclaw/test/*doctor*` and `plugins/codexclaw/test/*reset*`

Exact behavior:
- `cxc doctor` emits PASS/WARN/FAIL sections with evidence paths, command output,
  and actionable next steps.
- Doctor checks plugin manifest shape, `.mcp.json` optional server shape, skill
  frontmatter/policy files, hook registration where present, agent config where
  present, generated-state path health, and Codex availability.
- Doctor may delegate Codex runtime health to `codex doctor`, but codexclaw
  plugin health remains self-checked.
- `cxc reset --state` removes only codexclaw PABCD/session state.
- `cxc reset --generated` removes generated codexclaw files that are documented
  as regenerable.
- `cxc reset --all` composes the safe codexclaw scopes and must still avoid
  deleting user source files or Codex global config.
- Reset is constrained to the local `.codexclaw/` subtree and prints the exact
  removed-path summary after the scoped cleanup; no dry-run flag is shipped in
  MVP.

Must-NOT-Have:
- No deletion of Codex global config except by an explicit documented delegate.
- No bare "reinstall" diagnosis without evidence.
- No analytics/telemetry upload.
- No reset of other plugins, worktrees, or project files.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: implement command parser entries, doctor probes, reset scope resolver, and
  tests for safe path boundaries.
- A: reviewer focuses on destructive-operation safety and evidence quality.
- B: write commands, fixtures, and stdout shape tests; wire `cxc --help`.
- C: run node:test for doctor/reset and capture CLI stdout for a sample install.
- D: done = `cxc doctor` and `cxc reset --state` run locally and prove safe scope.

## Acceptance (1-3 testable criteria)
1. `cxc doctor` reports PASS/WARN/FAIL with evidence for each probe.
2. `cxc reset --state` deletes only codexclaw state paths in tests.
3. `cxc reset --all` never touches any non-codexclaw path.

## QA channel (node:test path / CLI stdout / tmux / data dump)
- node:test for report shape, path boundary checks, and reset fixture cleanup.
- CLI stdout snapshots for `cxc doctor` and `cxc reset --state`.
