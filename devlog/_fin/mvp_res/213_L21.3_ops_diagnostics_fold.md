# L21.3 (Decade 213) -- Ops Diagnostics Fold

## Goal (one slice)
Fold useful omo diagnostics and operations patterns into codexclaw without
adopting teammode, analytics telemetry, or git-bash MCP. The output enriches
`cxc doctor`, `cxc reset`, and packaging smoke tests.

## Why now / dependencies
- Upstream: L20.3 owns `cxc doctor` and `cxc reset`; L21 decides which omo ops
  pieces they absorb.
- Downstream: install verification and future bug-report workflows can reuse the
  doctor evidence format.
- This sub-pass prevents accidental adoption of heavy ops systems that do not fit
  MVP no-server constraints.

## Scope (decision-complete)
Files to add/edit in implementation:
- `plugins/codexclaw/src/cli/doctor.*`
- `plugins/codexclaw/src/cli/reset.*`
- `plugins/codexclaw/test/support/package-smoke-fixture.*`
- `plugins/codexclaw/docs/ops.md` or equivalent

Exact behavior:
- From `lcx-doctor`: adopt PASS/WARN/FAIL report shape, evidence-first checks,
  source/install/config/plugin/runtime drift comparison, and known-issue lookup
  pattern.
- From telemetry: adopt only local failure isolation. Hook or diagnostic failures
  write local JSONL evidence and do not block startup unless the failing component
  is required.
- From `test-support`: adopt JSON shape validators for package metadata, hooks,
  MCP config, plugin manifest, and skill policy files.
- From `lcx-report-bug` and `lcx-contribute-bug-fix`: defer to future follow-up
  skills. Only keep their bug routing and source ownership ideas as references.
- From git-bash: rely on Codex-native shell. Add a Windows reminder only if real
  shell failures are observed later.
- From teammode: do not adopt in MVP. Durable teams require Codex threads,
  `.codexclaw/teams`, title hygiene hooks, and worktree integration.

Must-NOT-Have:
- No PostHog or analytics endpoint in MVP.
- No teammode durable-thread orchestration.
- No git-bash MCP dependency.
- No doctor result that lacks evidence or a checked path/command.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: extend doctor/reset requirements, port package smoke validators, and mark
  deferred ops components in docs.
- A: reviewer checks privacy/no-analytics boundary and no-server fit.
- B: implement validators and enrich doctor sections with drift/known-issue slots.
- C: run node:test validators and capture sample `cxc doctor` stdout.
- D: done = ops value is folded into local diagnostics without adopting heavy
  orchestration systems.

## Acceptance (1-3 testable criteria)
1. `cxc doctor` has PASS/WARN/FAIL sections with evidence lines.
2. Smoke validators cover plugin manifest, MCP config, hooks, and skill policy.
3. No telemetry upload or teammode state path exists in MVP implementation.
