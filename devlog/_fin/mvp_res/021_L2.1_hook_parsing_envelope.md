# L2.1 (Decade 021) -- Hook Parsing and Output Envelope

Status: DONE
Cluster: 1 - Phase: 1 - Shorthand: cxc
Source-of-record: 018.2_pass2_P_plan.md; 018.3_state_transition_injection.md; STATUS.md

## Goal (one slice)
Parse Codex hook stdin defensively and emit the exact additionalContext envelope cxc needs.

## Why now / dependencies
L2 depends on the L1 state files but first needs trustworthy hook IO. Trigger logic and hybrid
injection are separate sub-passes built on these parsers and the envelope builder.

## Scope (decision-complete)
- Files added/edited:
  - `plugins/codexclaw/components/pabcd-state/src/parse.ts`
  - `plugins/codexclaw/components/pabcd-state/src/hook.ts`
  - `plugins/codexclaw/components/pabcd-state/src/cli.ts`
  - `plugins/codexclaw/components/pabcd-state/test/hook.test.ts`
- Parser functions:
  - `parseUserPromptSubmit(raw)`
  - `parseStop(raw)`
- Required snake_case input fields:
  - UserPromptSubmit: `hook_event_name`, `session_id`, `cwd`, `prompt`
  - Stop: `hook_event_name`, `session_id`, `cwd`
- Optional fields preserved:
  - `transcript_path`, `turn_id`, `model`, `permission_mode`, `stop_hook_active`,
    `last_assistant_message`
- Output function:
  - `buildContextOutput(eventName, ctx)`
  - wraps `hookSpecificOutput.hookEventName` and `additionalContext`
  - normalizes CRLF, trims, caps at 32k, and appends a trailing newline
- Must-NOT-Have:
  - No thrown parser errors on empty, corrupt, or wrong-shape input.
  - No stderr-only failure path that blocks Codex.
  - No camelCase input parsing.

## Acceptance (1-3 testable criteria)
- Empty, corrupt, wrong-event, or wrong-shape payloads return null or empty output.
- Non-empty context is emitted as one JSON line with `hookSpecificOutput.additionalContext`.
- Output always uses camelCase hook output keys and a trailing newline.
