## Reinforcement shape

1. Put the missing discipline back into the *thin injected surfaces*, not only `cxc-pabcd`: `cxc-orchestrate`, `cxc-loop`, and `cxc-goalplan` need short but load-bearing wording that survives when those are the only texts the agent sees.
2. Make every work-phase prompt name its required artifact: plan draft in P, audit receipt in A, implementation delta in B, verification bundle in C, closeout + next-step decision in D.
3. Upgrade continuation language from "the agent re-enters P" to "the agent must prove what remains, read durable state first, and may not shrink the objective to escape the loop."
4. Give `cxc-goalplan` a compact schema section: criterion fields, checkpoint bundle fields, steering kinds, and quality-gate object shape with example evidence slots.
5. Treat review and verification as blocker-producing gates in prompt text: timeout, ack-only, stale evidence, missing artifact, or weakened criteria mean continue / fix / re-run, not phase success.
6. Reuse jaw's stronger phrasing where possible: "do NOT ask questions in P", "a waiver is not silent ignore", "do not treat IDLE as the end", "match verification scope to claim scope", and "placeholder evidence is rejected".
