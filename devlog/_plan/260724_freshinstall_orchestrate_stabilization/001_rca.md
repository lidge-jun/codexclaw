# 001 — RCA: why fresh installs don't use orchestrate

Question from the user: on a fresh download the model barely uses
`orchestrate` — is it prompting or an install-script difference?

## Verdict: install/payload gap (not prompting, not an install script)

There is no install script at all — `codex plugin marketplace add` +
`codex plugin add` copy the payload verbatim. The payload is only
`plugins/codexclaw/` (`.agents/plugins/marketplace.json` →
`"path": "./plugins/codexclaw"`). The `cxc`/`codexclaw` bin is mapped from the
repo-root `package.json` (`bin/codexclaw.mjs`) which is OUTSIDE the payload.

- Local dev machine: repo checkout + npm bin link → `cxc` on PATH → every
  injected directive works → orchestrate gets used.
- Fresh marketplace install: no repo root, no npm link → `cxc: command not
  found` → the model tries the injected command once, fails, and silently
  abandons the FSM → "orchestrate 잘 안 씀".

The prompting layer itself fires correctly on fresh installs (hooks use
`node "${PLUGIN_ROOT}/components/*/dist/cli.js"`, PATH-independent). It is the
*content* of those injections that prescribes an unresolvable command.

## Evidence

1. Payload-only sandbox (`/tmp/cxc-fresh-sim2`: `cp -R plugins/codexclaw
   payload`, `env -i PATH=/usr/bin:/bin HOME=<sandbox>`):
   - `command -v cxc` → `NO-CXC-ON-PATH`.
   - Direct dist CLI works fully: `node payload/components/pabcd-state/dist/cli.js
     orchestrate status --session testsess` → exit 0,
     `session=testsess phase=IDLE ...`; after a simulated SessionStart hook,
     `orchestrate P --session freshsim` → exit 0 (`IDLE → P`); the P→A attest
     gate also fired correctly (demanded `planUnit`). The engine is 100%
     functional payload-only; only the command name is dead.
2. Full command-surface audit (Sol explorer, 2026-07-24): every actionable
   `cxc ...` instruction in the payload is verdict-(b) "fails on fresh
   install": `LOOP_ARM_DIRECTIVE` (hook.ts:343-363), `STOP_NEXT_COMMAND`
   (hook.ts:686-692,720-724), GOAL-IDLE-CONTINUE (hook.ts:787-803), goal-gate
   deny remedies (goal-gate src ~187,197), map/skill banners
   (map-affordance.ts:85,105,138,160), skill-search preamble (preamble.ts:18),
   recall injections (recall hook.ts:71-72,191-250), minds effort hint
   (minds.ts→dist:76), and every SKILL.md command line (loop, pabcd, dev,
   recall, repo-map, interview, search, remote refs). The ONLY PATH-independent
   surfaces are the hook JSONs / .mcp.json themselves and Discord `!cxc` chat
   commands.
3. Bonus defect found: `cxc scan evidence` (RESCAN_REINJECT_DIRECTIVE,
   hook.ts:968) has NO backing subcommand in either the root bin or
   pabcd-state cli.ts — broken even on local dev. A-round deepened this:
   `appendInterviewEvent` (state.ts:257) has zero production callers and
   nothing increments `tracker.scanRounds`, so the I→P scan soft-gate
   (interview.ts:297) is only passable via `override:true` today.
4. A-round addition: idle-edit.ts:33 is a further `cxc orchestrate status`
   emit site missed by the first audit sweep.

## Why "최초 실행 때" specifically

First-run sessions have zero transcript precedent of a working orchestrate
call. The model's first attempt fails with `command not found`, and nothing in
the payload offers the working fallback
(`node ${PLUGIN_ROOT}/components/pabcd-state/dist/cli.js orchestrate ...`),
so the FSM never arms and the Stop hook's continuation nudges keep prescribing
the same dead command. On the dev machine the same directives succeed, which
is why the problem is invisible locally.

## Fix direction (detailed in 010)

1. Ship a payload-resident dispatcher `plugins/codexclaw/bin/cxc.mjs`
   (thin delegator over component dist CLIs, mirroring root
   `bin/codexclaw.mjs`).
2. Resolve the command name at emit time: hooks detect whether `cxc` is on
   PATH; if not, rewrite emitted `cxc ` prefixes to
   `node "<payloadRoot>/bin/cxc.mjs" ` so every injected directive is
   copy-paste runnable on any install.
3. SessionStart banner announces the resolved invocation once, covering the
   static SKILL.md surfaces ("wherever docs say `cxc`, use ...").
4. Add the missing `scan record` subcommand (double write: ledger event +
   tracker.scanRounds) — referencing an existing surface is impossible, none
   exists (A-round H4). Full spec in 010.
