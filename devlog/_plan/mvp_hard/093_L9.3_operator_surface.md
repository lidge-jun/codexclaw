# L9.3 / 093 — Operator Surface: `cxc subagents` + `cxc provider`

Status: DONE (shipped + tested) · 2026-06-30 · mvp_hard loop L9.3 · class C2

> Closes L9.3 from the 090 slice map: the root CLI printed a Phase-2 placeholder
> for both `subagents` and `provider`. Now both are real, read/write or read-only.

## Gap

`bin/codexclaw.mjs` had:

```
case "subagents":
case "provider":
  console.log("... is a Phase 2 command ...");
```

So an operator could not inspect or change per-role model/prompt config from the
terminal, even though the store, MCP server, and GUI all already roundtrip
`.codexclaw/subagents.json`.

## Implementation

- New `components/subagent-config/src/cli.ts`:
  - `subagents` (or `list`) — print all role configs (JSON).
  - `subagents get <role>` — print one role config.
  - `subagents set <role> --mode default|model [--model <id>] [--prompt <text>|--clear-prompt]`
    — write through `setRole` (store owns `validateRolePatch` + atomic write).
  - pure `parseSubagentsArgs` + `runSubagents`; zero third-party deps.
- `bin/codexclaw.mjs` routes `subagents` -> the new CLI and `provider` -> the
  provider-bridge `detect` path (read-only `ocx status`, never `ensure`/`sync`).

This shares ONE source of truth: terminal, dashboard, and MCP all read/write the
same `.codexclaw/subagents.json`, which the L9.1 spawn-wrapper then consumes.

## Acceptance

- `subagents` lists all three roles; `get`/`set` roundtrip the store (proven by
  cli.test.ts + a live `cxc subagents` smoke run).
- invalid role / bad `--mode` / model-mode-without-id are surfaced as non-zero errors.
- `--prompt`/`--clear-prompt` set and clear the override.
- `cxc provider` prints the read-only ocx status line; it never mutates config.

## Verification

## Out Of Scope

- No new roles; the CLI operates on the existing explorer/reviewer/executor set.
