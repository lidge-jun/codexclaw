# L13.1 / 131 — WP2 Design: scan-evidence + I→P soft-gate (D2')

Status: P (plan) · 2026-06-30 · parent [130](130_L13_truthfulness_interview_impl.md)

> A-gate revision (2026-06-30, Boyle/FSM + Ohm/testability audit): the first draft had 5
> blockers. Revised design below supersedes the original sections. Key fix: the soft-gate
> CANNOT sit after `canEnter`, because `canEnter("P")` hard-rejects I→P when
> `flags.interview` is false (fsm.ts:44-50) — it would pre-empt the advisory/override path.

Root cause (Ontologist, 2-round rescan): `isInterviewReady` is a predicate over tracker
SHAPE/contents; "a contradiction scan actually ran" is an EVENT with no schema home, and
`fsm.ts` I→P only checks `flags.interview`. So "warn when no scan ran" had no state to read.

## Design

### 1. Scan-evidence in the tracker (hot cache)
`interview.ts` `InterviewTracker` gains:
- `scanRounds: number` — count of contradiction scans recorded this interview (default 0).
- `lastScanRoundId: number` — `roundId` at the most recent scan (default 0).
`normalizeInterview` + `reconstructInterview` carry both (default 0; clamped via `roundIdNum`).

### 2. Durable scan evidence (source of record)
New `.codexclaw/interviews/<sessionId>.jsonl`, append-only, written by `appendInterviewEvent`:
- events: `scan_started` | `scan_completed` | `rescan_completed`
- fields: `{ ts, sessionId, event, roundId, contradictionCount, highContradictionCount }`
The hot `scanRounds` counter is a cache of `scan_completed`/`rescan_completed` events.

### 3. Readiness = data-shape AND scan-evidence
`isInterviewReady` adds: `tracker.scanRounds >= 1` (a scan must have run at least once).
This keeps the existing single-source derive (`state.ts:94-96`) intact — `flags.interview`
still derives from the tracker, now including scan-evidence.

### 4. I→P soft-gate (human path) — REVISED to be reachable
The blocker: `canEnter("P")` (fsm.ts:44-50) hard-rejects I→P when `flags.interview` is false,
and `flags.interview` is DERIVED from `isInterviewReady` (state.ts:93). So a soft-gate placed
after `canEnter` can never run. Fix: treat `interview` exactly like the existing overridable
gate flags `auditPassed`/`checkPassed`, which `applyHumanTransition` PRE-FLIPS before calling
`canEnter` (orchestrate-apply.ts:103-117). The human override pre-flips `flags.interview=true`,
which is the explicit "I accept the risk" assertion.

`evaluateInterviewGate(tracker)` → `{ ready, scanRan, highContradictionCount, warnings[] }`
(pure, in interview.ts). In `applyHumanTransition`, ONLY for the `from==="I" && to==="P"` edge:
- ready → advance normally (canEnter passes because derived `flags.interview` is true).
- NOT ready AND no override → return `{ ok:false, reason:<advisory warning text> }` BEFORE
  canEnter, so the human sees a clear advisory (not the generic adjacency error). Advise-block.
- NOT ready AND `attest.override===true` → pre-flip `flags.interview=true`, advance, and emit a
  ledger entry with `actor:"human"`, `override:true`, `scanEvidence:{scanRounds,highContradictionCount}`.
The agent CLI path (`transition()`/`validateAttest`) is unchanged (stays hard-gated by attest).

### 5. Typed override signal (attest.ts) — REVISED
Do NOT use a `did.startsWith("override:")` heuristic (collision-prone in a free-form field).
Extend `Attestation` with an explicit optional `override?: boolean`, and update `coerceAttest`
to carry it (`typeof rec.override === "boolean"`). This is the typed control signal.

### 6. LedgerEntry audit fields (state.ts)
`LedgerEntry` gains optional `actor?: "human" | "agent"`, `override?: boolean`,
`scanEvidence?: { scanRounds: number; highContradictionCount: number }`. All optional →
backward compatible; `appendLedger` JSON.stringifies as-is; existing readers ignore extras.

### 7. goal-mode boundary — REVISED (claim narrowed)
Correction: goal mode suppresses only I-trigger injection + `request_user_input` (goal-gate.ts);
a human typing `orchestrate p` from an existing I state STILL reaches `applyHumanTransition`.
So the soft-gate is goal-mode-AGNOSTIC: it operates purely on tracker state and is harmless in
goal mode (it just enforces scan-evidence like anywhere else). The design no longer claims "not
applied in goal mode". Test asserts the soft-gate logic is independent of goal status, and the
existing goal-gate/​I-suppression tests remain green (separate surfaces).

### 8. JSONL reset scope (reset.ts)
`.codexclaw/interviews/<sessionId>.jsonl` must be cleaned by `cxc reset`. Add the `interviews/`
dir to the `--state` scope (session-scoped working state) so reset does not leave stale scan
ledgers. Align naming with the existing `.codexclaw/interview*` handling in reset.ts.

## Acceptance (C) — unit-testable assertions
1. scan-not-run: `isInterviewReady(maxedTracker with scanRounds:0)` === false.
2. scan-ran + clean: `isInterviewReady(maxedTracker with scanRounds:1)` === true.
3. advise-block: `applyHumanTransition(at("I"), "P")` with non-ready tracker → `{ok:false}` and
   reason contains the advisory warning (not the generic adjacency message).
4. override: `applyHumanTransition(at("I"), "P", {from:"I",to:"P",did:"...",override:true})` →
   `ok:true`, `state.phase==="P"`, and `ledger.override===true` && `ledger.actor==="human"` &&
   `ledger.scanEvidence` present.
5. legacy compat: a persisted all-max tracker WITHOUT scan fields reconstructs with scanRounds:0
   → `flags.interview` derives false (no silent ready); existing fixtures updated to add scan
   evidence where they intend "ready".
6. JSONL: `appendInterviewEvent` writes parseable scan events; a fresh read counts them; `cxc
   reset --state` removes `.codexclaw/interviews/`.
7. coerceAttest carries `override:true` through; ignores non-boolean.

(Process gates, run at C, not unit assertions: full suite green, build idempotent, diff clean.)
