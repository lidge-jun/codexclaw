## P1/P2 Findings — Subagents / Models / OMO Roles

### P1.4 Replay/idempotency rules should stay closer to codexclaw than jawcode

jawcode has useful workflow state and ledger machinery, but its replay/idempotency model has
several gaps that codexclaw should not copy:

- jawcode `UserPromptSubmit` skill activation writes state on keyword match without codexclaw-style
  `injectedTurns` + transcript-tail marker guards.
- jawcode goal mode appends `mode_change` entries directly to session JSONL outside its
  `SessionManager`, which can race live session persistence.
- jawcode goal ledger event IDs are random by default, so replayed checkpoint/steer attempts can
  double-log unless the current-state guard happens to short-circuit.
- jawcode goal mutations commonly write the plan JSON and then append the ledger as separate
  operations, leaving cross-file transaction gaps.

Evidence:
- codexclaw idempotency:
  `/Users/jun/Developer/new/700_projects/codexclaw/plugins/codexclaw/components/pabcd-state/src/hook.ts:175`,
  `/Users/jun/Developer/new/700_projects/codexclaw/plugins/codexclaw/components/pabcd-state/src/transcript.ts:37`.
- jawcode activation write path:
  `/Users/jun/Developer/new/700_projects/jawcode/packages/coding-agent/src/hooks/native-skill-hook.ts:171`,
  `/Users/jun/Developer/new/700_projects/jawcode/packages/coding-agent/src/hooks/skill-state.ts:327`,
  `:399`.
- jawcode direct session append:
  `/Users/jun/Developer/new/700_projects/jawcode/packages/coding-agent/src/jwc-runtime/goal-mode-request.ts:156`.
- jawcode ledger append:
  `/Users/jun/Developer/new/700_projects/jawcode/packages/coding-agent/src/jwc-runtime/goal-engine.ts:150`,
  `/Users/jun/Developer/new/700_projects/jawcode/packages/coding-agent/src/jwc-runtime/state-writer.ts:432`.
- jawcode write-plan-then-ledger examples:
  `/Users/jun/Developer/new/700_projects/jawcode/packages/coding-agent/src/jwc-runtime/goal-engine.ts:650`,
  `:1272`.

Backlog:
- For L3/L7, derive stable operation IDs for transition/checkpoint/steering events from
  `(sessionId, turnId, command, from, to)` or an explicit idempotency key.
- Keep `.codexclaw` workflow state separate from transcript mutation.
- Treat JSON snapshot write + ledger append as a documented two-step with recovery semantics,
  or write a pending/committed pair into the ledger.
