LOCKED 조타 원칙 2개 (000_INDEX에서 계승)

  • 역할을 늘리지 않는다. omo는 역할을 발명해(librarian/planner/gate-reviewer/qa-executor) 전문화한다. codexclaw는 3개 base role에 $cxc-* 스킬을 붙여 전문화한다. Plato의 역할 분할 제안은 명시적으로 기각.
  • host-native 경계. cli-jaw/omo 기능을 "격차"로 부르기 전에 Codex에서 누가 소유하는지부터 묻는다. goal mode / cwd / 스킬 발견 / hook dispatch / 메모리는 host-native — codexclaw가 다시 만들 게 아니다. 빠진 cxc 명령은 대개 격차가 아니라 호스트 책임이다.

강한 비목표 (structure/00_philosophy.md §2)

  • LSP 데몬 / codegraph MCP / 검색 서버 없음 (090).
  • SessionStart 자동 업데이트 / 텔레메트리 / 프로비저닝 없음 — provider bridge는 detect-only 유지.
  • 신규 서브에이전트 역할 없음 — 대신 스킬 첨부 (020).
  • goal-DB 쓰기는 금지가 아니라 게이트됨 (2026-07-01 수정) — 런타임은 thread/goal/set을 노출하므로, codexclaw는 인터뷰 freeze 승인 경계에서만 호스트 goal을 set하고 루프 중 self-arm은 금지. 분해 상태(workPhases/tasks/criteria)는 여전히 프로젝트 로컬 .codexclaw/goalplan.json (030).
