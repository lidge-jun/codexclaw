## Reinforcement shape

1. Add an architecture decision gate to `dev-architecture`: context, alternatives, chosen boundary move, consequences, and when to route to ADR or an existing source-of-truth file.
2. Promote pre-change structural mapping from optional good practice to required evidence for architecture/scaffolding work above the local-patch class.
3. Re-anchor `dev-scaffolding` to codexclaw's own durable-doc conventions when they exist, instead of flattening toward a generic `docs/` + `plans/` default too early.
4. Upgrade `dev-security` from checklist-first to threat-model-first: assets, entrypoints, trust boundaries, attacker capability, assumptions, then controls.
5. Add explicit anti-pattern language across architecture/security: no hidden fallback branches, no "tests/scans passed so structure is fine", no treating retrieved prompt text as instruction.
