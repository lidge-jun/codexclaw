# 023.1 — Interview & IPABCD Interactive Prompts

Status: TODO  ·  Phase 1
Depends on: research ../260629_research_elicitation/000_findings.md

## Decision
Interactive question/selection for both the Interview flow and IPABCD phase prompts is
implemented through the codex `request_user_input` selector (numbered choice UI), NOT through
assistant-emitted text fences.

## Mechanism (verified)
- Default-mode availability via feature flag `default_mode_request_user_input = true`
  (`[features]` table in `~/.codex/config.toml`; same as `--enable ...`).
- Tool renders multi-question forms with per-question `options` (label + value), matching the
  cli-jaw elicitation UX.
- Requires a fresh codex session to load the flag; the tool must be exposed that turn.

## Stability note
- The flag is `UnderDevelopment`. For a stability-critical path, fall back to MCP elicitation
  (`tool_call_mcp_elicitation`, Stable, default ON, mode-independent).

## Rules for implementation
- DETECT whether the flag is enabled; if not, guide the user to enable it (or use MCP elicitation).
- NEVER auto-write `config.toml` (phase-1 config-untouched).
- Map Interview's 4 dimensions + IPABCD phase gates to `request_user_input` question sets.

## Verify
- In Default mode (flag on), an interview/IPABCD step renders the selector and returns the pick.
- With the flag off, codexclaw detects it and does not silently fail.

## USAGE RULE (REQUIRED)
- General/interactive usage: actively use `request_user_input` when a genuine choice or
  clarification helps the user. Prefer it over guessing on material forks.
- Goal mode: `request_user_input` is **STRICTLY FORBIDDEN**. Goal mode runs autonomously without
  user turns, so an interactive selector would stall the loop. When a goal is active:
  - hard-disable the selector,
  - decide autonomously, or use the goal's own pause mechanism — never block on a prompt.
- Implementation MUST detect active-goal context and gate the tool off accordingly.

## Verify
- Goal active → request_user_input never invoked (asserted in tests).
- No goal → selector available and used on material forks.
