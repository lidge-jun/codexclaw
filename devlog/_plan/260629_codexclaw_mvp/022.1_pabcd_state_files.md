# 022.1 — PABCD State File Schema (REV: session-scoped per 016)

Status: TODO  ·  Phase 1

## Scoping decision (016, Finding C)
codex is PER-SESSION (not per-instance like cli-jaw). State is keyed by `sessionId` so parallel
sessions in one repo do not clobber each other. Mirrors omo `<cwd>/.omo/<comp>/<sid>-<aid>.json`.

## Files (per working tree, gitignored)
- `.codexclaw/sessions/<sanitize(sessionId)>.json` — per-session phase state.
- `.codexclaw/ledger.jsonl` — SHARED append-only audit; each entry tagged with `sessionId`.

## sessions/<sid>.json (draft)
```json
{
  "phase": "I",
  "sessionId": "<from hook payload session_id>",
  "slug": "260629_codexclaw_mvp",
  "updatedAt": "2026-06-29T13:00:00Z",
  "flags": { "interview": true, "auditPassed": false, "checkPassed": false },
  "supersededBy": null
}
```
Phases: `I` Interview, `P` Plan, `A` Audit, `B` Build, `C` Check, `D` Done.

## ledger.jsonl (append-only, shared)
```json
{"ts":"...","sessionId":"...","from":"P","to":"A","reason":"plan approved","evidence":"..."}
```

## Derived predicates (pure functions, mirror cli-jaw)
- `isAuditGateOpen(state)` — A may start only after P produced a plan.
- `isBuildGateOpen(state)` — B only after A passed.
- `isDone(state)` — all gates passed + D recorded.

## sessionId source
- Pass 1: state module takes `sessionId` as a parameter (testable in isolation).
- Pass 2: UserPromptSubmit/Stop hook reads `session_id` from the codex hook payload and passes it in.

## Verify
- Functions pure + unit-tested.
- Corrupt/missing file → safe default (no crash).
- Two sessionIds in one cwd keep independent state (regression test).
