## Activation mechanism (resolves config-untouched tension with 027)
- Official path: `codex features enable <name>` writes `[features].<name> = true` into
  `~/.codex/config.toml` (equivalent to `-c features.<name>=true`).
- This is a SANCTIONED install action, not silent mutation. So 027 is revised:
  - 027 "config untouched" now means: codexclaw does NOT arbitrarily edit unrelated config; it ONLY
    flips the specific, documented feature flags it needs, via the official `codex features` CLI,
    and only at install/enable time with user awareness.
  - codexclaw should back up config.toml before flipping, and `codexclaw uninstall` should revert
    exactly the flags it set (mirror opencodex's `ocx stop`/restore discipline).
