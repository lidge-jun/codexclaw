## 기존 자산 (확인됨)
- `032` `.codexclaw/subagents.json`: `roles.<role> = { mode: default|model, model, promptOverride }`.
- `033` 모델 카탈로그 = ocx synced models + main(default) = n+1. ocx 부재 시 [main]만.
- `025` role.rs: per-role 모델 오버라이드가 session-flag precedence로 동작(검증됨) → 근거 확보.

## 설계: 5-Mind를 role로 등록
`.codexclaw/subagents.json`의 `roles`에 인터뷰 Mind 5종을 추가:
```json
{
  "roles": {
    "mind-contrarian":  { "mode": "model",   "model": "<strong-reasoning>", "promptOverride": "<contrarian spec>" },
    "mind-socratic":    { "mode": "default", "model": null,                 "promptOverride": "<socratic spec>" },
    "mind-ontologist":  { "mode": "default", "model": null,                 "promptOverride": "<ontologist spec>" },
    "mind-evaluator":   { "mode": "model",   "model": "<strong-reasoning>", "promptOverride": "<evaluator spec>" },
    "mind-simplifier":  { "mode": "model",   "model": "<cheap-fast>",       "promptOverride": "<simplifier spec>" }
  }
}
```
- 모델값은 `033` 카탈로그에서 고른 것만 허용(검증: 카탈로그 외 모델 → 거부 또는 default fallback).
- promptOverride = 080 §3의 Mind role prompt (출력 = 모순 only, {dimension,contradiction,severity,evidence}).

## 디스패치 2-tier (080 spawn_agent 가드 재사용)
- spawn_agent 가용 → `spawn_agent({ agent_type: "mind-contrarian", model: <resolved>, message: "..." })`.
  role의 model이 `mode:"model"`이면 그 모델로, `default`면 main 모델로.
- spawn_agent 불가(구버전 codex) → 자연어 위임 fallback: 메인이 해당 Mind 프롬프트를 인라인 순차 실행
  (이 경우 per-Mind 모델 지정은 무력화 — main 모델 단일. 이 degrade를 명시).

## ❓확정 필요 (jun) — 아래는 모순-도출 dry-run 결과와 함께 질문 예정
- 기본 모델 배정안: Contrarian/Evaluator=강추론, Simplifier=경량, Socratic/Ontologist=default? (비용·지연 trade)
- 5종 서로 다른 모델 동시 spawn 시 비용/지연 상한 — 동시 spawn 수 cap(080은 max 2 동시) 유지?
- 설정 모델이 카탈로그에서 사라졌을 때(ocx 동기화 변동) fallback 규칙: default로 강등 vs 에러.
- per-Mind 모델 = 결정성 저하 우려. 인터뷰 재현성이 필요한가(테스트 시 고정 모델 강제?).
