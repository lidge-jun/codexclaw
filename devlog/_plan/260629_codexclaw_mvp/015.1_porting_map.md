## Ground truth: codex subagent = "agent role"
- A role file is loaded with the SAME machinery as `config.toml` (role.rs:1-7).
- Role chosen at spawn via `agent_type`; the multi-agent tool handler owns orchestration.
- Role layer = session-flag precedence: overrides persisted config, BUT caller's
  `model_provider`/`service_tier` stay sticky unless the role sets them → per-role model override
  is supported (basis for Phase 2 multi-model).
- Parser: `AgentRoleConfig` + `parse_agent_role_file_contents`. Built-ins: explorer, awaiter.
- Spawn tools (from omo usage): `multi_agent_v1.spawn_agent / wait_agent / close_agent`,
  `fork_context: false` default.
