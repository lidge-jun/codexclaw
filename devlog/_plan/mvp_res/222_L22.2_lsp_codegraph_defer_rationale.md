# L22.2 (Decade 222) -- LSP and Codegraph Defer Rationale

Status: ANALYZED
Cluster: 3 · Phase: expansion · Shorthand: cxc
Source-of-record: 220_L22_code_intelligence.md; 260629_codexclaw_mvp/130_code_intelligence.md; 090.1 J-16; jun 130-defer resolution

## Goal (one slice)
Record the resolved non-goal: LSP and codegraph are explicitly deferred out of
MVP. This is no longer blocked on a decision. The decision is to keep core
codexclaw lightweight and adopt only ast-grep for MVP code intelligence.

## Why now / dependencies
- Upstream: L22.1 adopts ast-grep, so this sub-pass draws the boundary around
  what code intelligence does not include.
- J-16 already requires lsp/codegraph to be isolated as separate install
  extensions if they ever return.
- The jun 130-defer resolution selects Option A: ast-grep only; lsp/codegraph are
  post-MVP due to daemon, language-server, indexing, and workspace side effects.
- Downstream: L19 skill-hub must not catalog lsp/codegraph as MVP on-demand core
  skills, and `cxc doctor` must not fail core health because they are absent.

## Scope (decision-complete)
Files to add/edit in implementation:
- `plugins/codexclaw/docs/code-intelligence.md` or equivalent non-goal note
- `plugins/codexclaw/skills/skill-hub/references/catalog.md` after L19 unblock
- `plugins/codexclaw/.mcp.json` must remain free of core lsp/codegraph entries
- `plugins/codexclaw/src/cli/doctor.*` should treat lsp/codegraph as absent by
  design, not missing dependencies

Exact behavior:
- LSP is deferred because it requires daemon/proxy wiring, external language
  servers, post-edit diagnostics hooks, setup workflows, and potentially mutating
  rename operations.
- Codegraph is deferred because it requires external binary/provisioning, session
  start init/sync, background indexing, optional MCP degradation, and workspace
  side effects.
- Neither system is added to the core plugin manifest, default `.mcp.json`,
  default hooks, or default doctor required checks.
- Future lsp/codegraph work must be a separate install extension or feature flag,
  with its own doctor checks, privacy/workspace-side-effect notes, and explicit
  user activation.
- Core docs should recommend native shell/`rg` for text search, ast-grep for
  structural search/codemods, project test/typecheck for verification, and future
  extension only for semantic IDE-grade navigation.

Must-NOT-Have:
- No `lsp` or `codegraph` MCP server in core `.mcp.json`.
- No detached daemon, socket server, session-start indexer, or language-server
  installer in MVP.
- No doctor FAIL because lsp/codegraph are missing.
- No catalog entry implying lsp/codegraph are shipped MVP skills.
