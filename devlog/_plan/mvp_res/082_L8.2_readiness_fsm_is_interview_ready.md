# L8.2 (Decade 082) -- Readiness FSM and isInterviewReady

Status: FROZEN
Cluster: 1 - IPABCD Interview completion - Phase: 1 - Shorthand: cxc
Source-of-record: 080, 022.2, 022.3

## Goal (one slice)
Implement the readiness predicate and keep the P entry gate tied to the derived interview-ready flag.

## Why now / dependencies
L8.1 must define `InterviewTracker`. This sub-pass unblocks L10 freeze by giving the main session one predicate for deciding whether I can close.

## Scope (decision-complete)
- Files to add/edit: `plugins/codexclaw/components/pabcd-state/src/state.ts`, `plugins/codexclaw/components/pabcd-state/src/fsm.ts`, `plugins/codexclaw/components/pabcd-state/test/state.test.ts`, `plugins/codexclaw/components/pabcd-state/test/fsm.test.ts`.
- Add `isInterviewReady(tracker: InterviewTracker | null): boolean`.
- Return false for null, malformed, or incomplete trackers.
- Return true only when all four dimensions have level `max`, `contradictions[]` is empty, and every assumption has `recorded: true`.
- `assumptions[]` may be non-empty; the invariant is recorded visibility in `## OPEN ASSUMPTIONS`, not zero assumptions.
- Preserve `canEnter("P")` interface: it still checks `state.flags.interview === true`.
- Main-session behavior sets `flags.interview = isInterviewReady(state.interview)`; user transition requests cannot override a false predicate.
- Must-NOT-Have: no new phase enum, no direct P gate on raw tracker inside `fsm.ts`, no relaxed readiness for high-severity contradictions.

## IPABCD micro-cycle
- I: Each interview round recalculates readiness after triage and plan edits.
- P: Add predicate tests first, then document the `flags.interview` derivation near the FSM gate.
- A: Reviewer checks R-H/R5 drift: contradictions must be empty and unrecorded assumptions must block.
- B: Implement `isInterviewReady()` with explicit dimension iteration and no trust in `tracker.ready` alone.
- C: Run targeted state and FSM node tests.
- D: Done = complete tracker can enter P through `flags.interview`; incomplete tracker cannot.

## Acceptance (1-3 testable criteria)
1. All-max dimensions plus empty contradictions plus recorded assumptions returns true.
2. Any non-max dimension, any contradiction, or any `recorded:false` assumption returns false.
3. `canEnter("P")` remains backward-compatible and blocks when `flags.interview` is false.
