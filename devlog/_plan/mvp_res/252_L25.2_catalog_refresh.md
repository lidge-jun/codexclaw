# L25.2 (Decade 252) -- Catalog Refresh

Status: PLANNED
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 250_L25_model_catalog.md, 260629_codexclaw_mvp/033_model_catalog.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Expose a refreshable catalog read path so GUI and CLI consumers can re-check the
native-plus-ocx model list without writing selected role config.

## Why now / dependencies
- Upstream: depends on L25.1 catalog source and L23 provider status.
- Downstream: lets L27 load current choices and lets L28 verify S7 with fixture
  refreshes.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/components/subagent-config/src/catalog.ts`
  - `plugins/codexclaw/components/subagent-config/src/mcp.ts`
  - `plugins/codexclaw/components/subagent-config/test/catalog.test.ts`
  - `plugins/codexclaw/components/subagent-config/test/mcp.test.ts`
- Exact behavior:
  - Provide an explicit catalog read/refresh API.
  - Refresh re-runs provider status/catalog reads, but never calls arbitrary
    provider commands from the GUI.
  - Return status for native-catalog, ocx-active, and unsupported ocx catalog.
  - Do not persist selected role models; GUI save remains L24.
- Must-NOT-Have:
  - No ocx auto-ensure from React.
  - No hidden writes to `.codexclaw/subagents.json`.
  - No stale success if ocx disappears between reads.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: add read/refresh surface with status codes, timestamps if already available
  from provider status, and deterministic fixture tests.
- A: audit angle = "can refresh lie about current provider state?" Reviewer
  checks ocx present-to-absent and absent-to-present fixture transitions.
- B: implement refresh method, API/MCP exposure, error status propagation, and
  tests for transition cases.
- C: run node:test for refresh; dump catalog JSON before and after fixture state
  changes.
- D: done = GUI can refresh model choices and show current native/ocx state.

## Acceptance (1-3 testable criteria)
1. Refresh after ocx disappears returns native-catalog state.
2. Refresh after ocx appears returns native-plus-ocx fixture entries.
3. Refresh never writes role selections or prompt overrides.

## QA channel (node:test path / CLI stdout / tmux / data dump)

- node:test path: `plugins/codexclaw/components/subagent-config/test/mcp.test.ts`
- Data dump: catalog refresh transition JSON.
