# L10.2 (Decade 102) -- Auto-Mode Assumptions

Status: FROZEN
Cluster: 1 - IPABCD Interview completion - Phase: 1 - Shorthand: cxc
Source-of-record: 080.2, ouroboros 040, elicitation 000_findings

## Goal (one slice)
Define severity triage so low and medium contradictions become recorded assumptions while high contradictions go to the user.

## Why now / dependencies
L8 defines `Assumption.recorded` and readiness. L9 supplies normalized severity. L10.1 supplies the question path for high severity.

## Scope (decision-complete)
- Files to add/edit: `plugins/codexclaw/components/pabcd-state/src/state.ts`, `plugins/codexclaw/components/pabcd-state/src/hook.ts`, `plugins/codexclaw/components/pabcd-state/test/state.test.ts`, `plugins/codexclaw/components/pabcd-state/test/hook.test.ts`.
- Low severity becomes a conservative recorded assumption.
- Medium severity becomes a conservative recorded assumption by default.
- High severity in manual interview must call `request_user_input` and cannot be safe-defaulted.
- Goal-mode backfill is different: user-only high gaps become high-severity assumptions requiring user review, because goal mode cannot ask.
- Auto-resolved contradictions must be removed from `contradictions[]` and pushed to `assumptions[]` with `recorded:true` only after they are written into `## OPEN ASSUMPTIONS`.
- After three consecutive auto-resolves, the next contradiction must be escalated to the user.
- Max-round closure supports `safe_default`, `ledger_only`, and `genuine-deadlock`; genuine deadlock is the only hard stop.
- Must-NOT-Have: no silent high-severity default in manual interview, no unrecorded assumption readiness, no subagent triage authority.

## IPABCD micro-cycle
- I: Main triages each contradiction before asking or editing.
- P: Add severity policy text and state tests for assumption readiness.
- A: Contrarian checks that high severity cannot disappear into auto-mode.
- B: Implement assumption transitions and rhythm-guard directive text.
- C: Run state and hook tests.
- D: Done = every contradiction exits as user-resolved fact or recorded assumption before readiness.

## Acceptance (1-3 testable criteria)
1. Low and medium contradictions are converted to recorded assumptions and removed from contradictions.
2. High contradictions are user questions in manual interview.
3. Three auto-resolves force the next contradiction to user escalation.
