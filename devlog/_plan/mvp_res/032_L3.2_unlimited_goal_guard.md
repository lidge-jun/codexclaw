# L3.2 (Decade 032) -- Unlimited Goal Guard

Status: DONE
Cluster: 1 - Phase: 1 - Shorthand: cxc
Source-of-record: 023_goal_convention_port.md; 023.2_pass3_P_plan.md; STATUS.md

## Goal (one slice)
Deny budgeted or extra-keyed `create_goal` tool calls so cxc goals stay unlimited.

## Why now / dependencies
The parser is available from L3.1. This sub-pass implements the actual policy while keeping goal-mode
interview hard-deny out of Phase 1.

## Scope (decision-complete)
- Files added/edited:
  - `plugins/codexclaw/components/pabcd-state/src/goal-gate.ts`
  - `plugins/codexclaw/components/pabcd-state/test/goal-gate.test.ts`
- Guard function:
  - `applyGoalBudgetGuard(payload)`
- Policy:
  - target tool name is exactly `create_goal`.
  - `tool_input` may contain only `objective`.
  - any extra key, including `token_budget`, is denied.
  - objective-only, empty-object, non-object, and unrelated tools pass through as empty output.
- Deny output:
  - `hookSpecificOutput.hookEventName = "PreToolUse"`
  - `permissionDecision = "deny"`
  - `permissionDecisionReason` explains objective-only unlimited goals.
  - `additionalContext` repeats the same warning.
- Must-NOT-Have:
  - No budget workaround.
  - No custom goal lifecycle state machine.
  - No deny for `update_goal`.
  - No goal-mode request_user_input hard deny in this shipped sub-pass.

- `create_goal` with `{ objective, token_budget }` returns a deny envelope.
- Any extra key on `create_goal` returns the same deny envelope.
