# L24.1 (Decade 241) -- subagents.json Schema

Status: PLANNED
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 240_L24_subagent_config_store.md, 260629_codexclaw_mvp/032_subagent_config_store.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Define the `.codexclaw/subagents.json` schema for explorer, reviewer, and
executor role model mode plus nullable prompt override.

## Why now / dependencies
- Upstream: depends on L5 role names and the L24 head decision that default mode
  must work without ocx.
- Downstream: gives L24.2, L25, L27, and L28 one stable config contract.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/components/subagent-config/src/store.ts`
  - `plugins/codexclaw/components/subagent-config/test/store.test.ts`
- Exact behavior:
  - Store path is `.codexclaw/subagents.json`.
  - Top-level shape is `roles.explorer`, `roles.reviewer`, `roles.executor`.
  - Each role has `mode`, `model`, and `promptOverride`.
  - `mode` is `default` or `model`.
  - Missing file resolves to defaults for all three roles.
- Must-NOT-Have:
  - No extra role names in Phase 2.
  - No ocx requirement for default-mode config.
  - No global Codex config mutation.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: encode schema types, defaults, validation messages, and normalization rules
  before wiring any writes or spawn behavior.
- A: audit angle = "can malformed config break subagent startup?" Reviewer
  checks invalid JSON, missing roles, invalid mode, and null handling.
- B: implement default factory, parser, validator, and normalized read return.
- C: run node:test store fixtures for missing, valid, partial, and malformed
  files.
- D: done = every consumer can read a validated config object with safe defaults.

## Acceptance (1-3 testable criteria)
1. Missing `.codexclaw/subagents.json` returns all roles in `mode: default`.
2. Invalid `mode` or unknown role shape produces a clear validation error.
3. `promptOverride` accepts string or null and never defaults to fabricated text.

## QA channel (node:test path / CLI stdout / tmux / data dump)
- node:test path: `plugins/codexclaw/components/subagent-config/test/store.test.ts`
- Data dump: normalized default config object.
