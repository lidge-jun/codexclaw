# L27.2 (Decade 272) -- 10100 Link Bar

Status: DONE
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 270_L27_gui_subagent_page.md, 260629_codexclaw_mvp/035_gui_subagent_page.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Add an opencodex dashboard link bar to the GUI that appears only when provider
status reports ocx detected.

## Why now / dependencies
- Upstream: depends on L23 provider status and L26 GUI shell.
- Downstream: gives L28 direct S9 evidence and keeps default-only users away
  from a broken `localhost:10100` link.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/gui/src/components/OcxLinkBar.tsx`
  - `plugins/codexclaw/gui/src/pages/Subagents.tsx`
  - `plugins/codexclaw/gui/src/api.ts`
  - GUI smoke fixture for ocx-present and ocx-absent states
- Exact behavior:
  - Query codexclaw provider status, not ocx directly from React.
  - Show link to `http://localhost:10100` only when ocx is detected.
  - Hide the link bar when ocx is absent, unsupported, or failing ensure.
  - Keep model selector usable in default-only mode.
- Must-NOT-Have:
  - No unconditional link to opencodex.
  - No automatic opening of `localhost:10100`.
  - No assumption that opencodex GUI assets are present locally.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: add provider-status API call, link-bar component, hidden/error states, and
  smoke fixtures for present/absent status.
- A: audit angle = "does link visibility track real provider status?" Reviewer
  checks absent, detected, ensure-failed, and unsupported-catalog states.
- B: implement component and integrate it into Subagents page or app shell.
- C: run GUI build; run browser smoke with ocx absent and fixture-present status.
- D: done = S9 passes with a gated link and no broken default-only experience.

## Acceptance (1-3 testable criteria)
1. ocx-present fixture shows `http://localhost:10100`.
2. ocx-absent fixture hides the link bar.
3. ensure-failed status does not show the link as active multi-provider state.

## QA channel (node:test path / CLI stdout / tmux / data dump)
- GUI build output from `plugins/codexclaw/gui`.
- Browser smoke check for ocx absent and ocx present fixtures.
- Provider status fixture dump used by the GUI.
