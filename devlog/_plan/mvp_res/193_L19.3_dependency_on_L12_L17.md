## Goal (one slice)
Document the exact blocking dependency that prevents the skill hub rewrite from
being implemented early: the hub catalog must be generated from the final L12-L17
skill set and its absorbed omo content, not from provisional plan names.

## Why now / dependencies
- L19 is a cataloging and exposure loop, so incorrect upstream names are user
  visible failures: hidden skills become unreachable if the catalog points at the
  wrong path.
- L12-L17 decide final surfaces for dev hub, `pabcd`, role-specific dev skills,
  GitHub delegation, design skills, review skills, security/devops/scaffold, and
  absorbed omo behavior.
- J-15 clarifies that 090+ docs are future implementation plans; live Phase-1
  outputs win where there is overlap.
