# L27.1 (Decade 271) -- Role Model and Prompt Page

Status: PLANNED
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 270_L27_gui_subagent_page.md, 260629_codexclaw_mvp/035_gui_subagent_page.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Build the Subagents GUI page for explorer, reviewer, and executor model
selection plus nullable per-role prompt override editing.

## Why now / dependencies
- Upstream: depends on L24 config persistence, L25 catalog refresh, and L26 GUI
  app shell.
- Downstream: provides the user-facing proof for L28 S7, S8, and S10.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/gui/src/pages/Subagents.tsx`
  - `plugins/codexclaw/gui/src/components/ModelSelect.tsx`
  - `plugins/codexclaw/gui/src/components/PromptOverrideEditor.tsx`
  - `plugins/codexclaw/gui/src/api.ts`
  - GUI tests or smoke fixtures under `plugins/codexclaw/gui/src/`
- Exact behavior:
  - Show explorer, reviewer, and executor rows or cards.
  - Populate model selectors from L25 catalog.
  - Keep default/main model selectable even when ocx is active.
  - Save model mode, selected model, and prompt override through L24 API.
  - Reload shows persisted settings.
- Must-NOT-Have:
  - No direct file writes from React.
  - No direct shell calls to ocx.
  - No prompt override that replaces unrelated system/dev-skill content.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: implement page layout, model selector, prompt editor, load/save states, and
  error/empty states around L24/L25 API shapes.
- A: audit angle = "does UI state roundtrip to spawn config?" Reviewer checks
  saved JSON and resolved spawn behavior.
- B: add components, API calls, save behavior, and smoke fixtures.
- C: run GUI build; save a reviewer model and prompt, then inspect
  `.codexclaw/subagents.json`.
- D: done = S8 and S10 are demonstrable from the GUI.

## Acceptance (1-3 testable criteria)
1. Saving a reviewer model persists and reloads with the same value.
2. Saving a prompt override persists as string or null.
3. The model selector always includes the default/main model.

## QA channel (node:test path / CLI stdout / tmux / data dump)
- GUI build output from `plugins/codexclaw/gui`.
- Browser smoke check for Subagents page save/reload.
- Data dump: `.codexclaw/subagents.json` after model and prompt edits.
