# L18.2 (Decade 182) -- Korean Search Intent Guard

Status: ANALYZED
Cluster: 2 (part B) · Phase: expansion · Shorthand: cxc
Source-of-record: 180_L18_unified_search_hub.md; 260629_codexclaw_mvp/120_unified_search_hub.md

## Goal (one slice)
Port the cli-jaw Korean "search" intent guard into the codexclaw `search` skill
so Korean lookup verbs route to external search when appropriate, not accidental
repository grep. The guard rewrites vague Korean requests into focused queries
and keeps original-page verification mandatory when the result matters.

## Why now / dependencies
- Upstream: L18.1 fixes the backend ladder that this guard dispatches into.
- The guard must exist before L19 catalogs `search` as an implicit-visible skill,
  because Korean trigger words are part of the skill exposure contract.
- Downstream: prevents future command-mapping and chat-search work from treating
  bare "검색" as local file search by default.

## Scope (decision-complete)
Files to add/edit in implementation:
- `plugins/codexclaw/skills/search/SKILL.md`
- `plugins/codexclaw/skills/search/references/query-rewrite.md`

Exact behavior:
- Trigger words include `검색`, `검색해`, `찾아봐`, `찾아줘`, `알아봐`, `웹검색`,
  and English equivalents such as `search`, `look up`, `latest`, `current`, and
  `news`.
- First classify the target: external/public/current info, programming docs,
  repository code/logs/config, or ambiguous.
- External/public/current info uses the L18.1 search ladder.
- Programming library/framework/API documentation should prefer official docs or
  the active documentation retrieval path when available, then source-open proof.
- Repository code/logs/config uses file search (`rg`, `rg --files`, codegraph or
  local tools when available) and must not be misrouted to web search.
- Ambiguous bare "검색" asks one short clarification rather than launching a
  repo-wide grep or a web search.
- Rewrite the user request into 1-3 focused keyword queries. Preserve anchor
  entities, source hints such as official/Naver/GitHub, dates, locale, and content
  type. Do not send the full natural-language sentence as the only query.
- Treat search results as candidate URLs. Open the original page for evidence
  when factual accuracy, recency, or source attribution matters.

Must-NOT-Have:
- No default repo-wide grep just because the Korean word "검색" appears.
- No answer from snippets alone when the user asks for latest/current facts.
- No dependency on `agbrowse`; query planning is encoded in the reference file.
- No hidden fallback between web, docs, and file search. Classification is stated.

## IPABCD micro-cycle
- I: not interview-bearing; ambiguity is resolved by one clarification question.
- P: copy the 8-rule guard shape from cli-jaw, replacing backend names with the
  codexclaw ladder and removing jaw-only provider names.
- A: reviewer checks that every Korean trigger either classifies cleanly or asks
  one clarification, and that docs/library queries do not bypass official docs.
- B: write the trigger block, classification table, rewrite template, and proof
  rule into the skill.
- C: dry-run three prompts: bare "검색", Korean current-info request, and Korean
  repository-file request. Confirm each routes differently.
- D: done = Korean lookup intent no longer silently means local grep or snippet
  answer.

## Acceptance (1-3 testable criteria)
1. `search/SKILL.md` includes the Korean trigger list and classification rules.
2. `query-rewrite.md` documents 1-3 focused query generation with anchor
   preservation.
3. A bare ambiguous "검색" path asks a clarification instead of acting.
