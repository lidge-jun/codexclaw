# L18.1 (Decade 181) -- Three-Tier Search Ladder

Status: ANALYZED

## Goal (one slice)
Define the codexclaw search ladder as exactly three Codex-native tiers: hosted
web search for discovery, Browser Use / Computer Use for proof, and explicit
subagent swarm for deep research. This sub-pass removes cli-jaw dead tiers while
preserving the core invariant: snippets discover URLs, opened sources prove facts.

## Why now / dependencies
- Upstream: L12-L17 establish the implicit dev discipline; L18 adds the search
  peer that L19 must list in the final implicit set.
- J-10 already resolved the contradiction between cli-jaw's 4-tier ladder and
  codexclaw's no-server runtime: progrok and hosted web-AI do not carry over.
- Downstream: L18.2 consumes this ladder for Korean intent routing; L18.3 maps
  omo ultimate-browsing and ultraresearch into helper/third-tier positions.

Exact behavior:
- Tier 1 is `web_search` or the current hosted web-search tool. It runs 1-3
  rewritten, focused queries and returns candidate URLs plus source metadata.
- Tier 1 never marks an answer sufficient solely from snippets, summaries, or
  search-result consensus. If the fact matters, the original source is opened.
- Tier 2 is Browser Use / Computer Use. Browser Use reads DOM, PDFs, screenshots,
  and rendered pages; Computer Use is reserved for browser chrome or OS UI that
  the in-app browser cannot reach.
- Tier 2 is the default proof tier. It should open the source page, confirm date,
  author/source identity, exact claim, and whether the page is primary evidence.
- Tier 3 is subagent swarm in ultraresearch mode. It is opt-in for broad, costly,
  or multi-source research and must be requested by the main agent, not auto-fired
  for ordinary latest/current lookups.
- The ladder text should name removed cli-jaw tiers explicitly as non-goals so
  future maintainers do not re-add progrok or web-AI by habit.

Must-NOT-Have:
- No progrok tier, Exa/Tavily/Perplexity provider promise, or hosted web-AI wait.
- No `agbrowse research plan` hard dependency. Query rewrite is prompt-side.
- No CloakBrowser or agent-browser vendoring in the core plugin.
- No claim that Tier 3 is durable; durability belongs to Phase 3 scheduler work.
