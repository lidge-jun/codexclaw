# L26.1 (Decade 261) -- Vite React Scaffold

Status: RESOLVED
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 260_L26_gui_scaffold.md, 260629_codexclaw_mvp/034_gui_scaffold.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Create the codexclaw GUI app shell using Vite + React so Phase 2 has an owned
web surface for subagent configuration.

## Why now / dependencies
- Upstream: depends on D4 that codexclaw ships its own GUI and L24/L25 API
  contracts for future data.
- Downstream: unblocks L26.2 serving, L27 Subagents page, and L28 GUI checks.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/gui/package.json`
  - `plugins/codexclaw/gui/index.html`
  - `plugins/codexclaw/gui/src/main.tsx`
  - `plugins/codexclaw/gui/src/App.tsx`
  - `plugins/codexclaw/gui/src/api.ts`
  - `plugins/codexclaw/gui/src/styles.css`
  - `plugins/codexclaw/gui/README.md`
- Exact behavior:
  - Use Vite + React, mirroring opencodex stack familiarity.
  - Use opencodex GUI only as a layout/structure reference; implement
    codexclaw-owned content and endpoints.
  - App shell loads without ocx and without existing subagent config.
  - API client talks to codexclaw APIs, not directly to ocx.
  - Document full `codexclaw gui` once if needed; examples use `cxc gui`.
- Must-NOT-Have:
  - No vendored opencodex source or copied components.
  - No TUI or install wizard.
  - No hidden dependency on `localhost:10100`.

## IPABCD micro-cycle
- I: not interview-bearing; Q-P2-1 resolved opencodex as layout/structure
  reference only.
- P: scaffold package, app root, API client stubs, styles, and README around the
  chosen GUI source policy.
- A: audit angle = "does the app stay owned by codexclaw and avoid ocx
  vendoring?" Reviewer checks imports, package deps, and file origins.
- B: create app shell, route container, empty state, and minimal API boundary.
- C: run GUI build and browser smoke for app shell with ocx absent.
- D: done = Vite app builds and loads as a codexclaw-owned dashboard shell.

## Acceptance (1-3 testable criteria)
1. `plugins/codexclaw/gui` builds with Vite + React.
2. App shell loads with no ocx and no `.codexclaw/subagents.json`.
3. Source tree contains no copied opencodex GUI files.

## QA channel (node:test path / CLI stdout / tmux / data dump)
- GUI build output from `plugins/codexclaw/gui`.
- Browser/console smoke check for app shell.

## Commit unit (one atomic conventional commit)
`feat(gui): scaffold vite react dashboard`

## Blocked-on (jun decision id, if any)
None. Q-P2-1 resolved: opencodex GUI is a layout/structure reference only; do
not copy, vendor, or import opencodex GUI files.

## Resolved (jun 2026-06-30)
Q-P2-1 resolved GUI reuse policy: codexclaw builds its own Vite/React GUI and
borrows only layout patterns from opencodex. opencodex GUI is API-driven (for
example, `gui/src/pages/Subagents.tsx` fetches `/api/*`), so codexclaw implements
new content and codexclaw endpoints while keeping acceptance criterion 3:
source tree contains no copied opencodex GUI files.
