# L2.5 (Decade 025 slot) -- R-12 decision: PostCompact / SubagentStop NOT required for MVP

Cluster: 1 (L2 hardening) - Phase: 1 - Shorthand: cxc

## Decision
codexclaw does NOT register PostCompact or SubagentStop hooks for the MVP. Rationale grounded in the
omo reference + the L2.4 mechanism we just shipped:

1. **PostCompact is not used by omo for directive injection.** omo registers PostCompact ONLY for
   cache resets — `post-compact-resetting-lsp-diagnostics-cache.json`,
   `post-compact-resetting-project-rule-cache.json`,
   `post-compact-resetting-git-bash-mcp-reminder.json`. Its directive-injection components
   (`ulw-loop` -> UserPromptSubmit+PreToolUse; `ultrawork` -> UserPromptSubmit) do NOT register
   PostCompact. So "inject on compaction" is not an omo pattern.
2. **L2.4 already covers the post-compaction idempotency hole R-12 worried about.** The transcript-tail
   stage-marker scan (`hasStageMarkerForPhase`) + context-pressure suppression (`isContextPressureTail`)
   make re-injection safe across turn_id churn and compaction without a dedicated PostCompact hook.
   Injection re-establishes naturally on the next UserPromptSubmit (codex fires it per user turn).
3. **SubagentStop has no codexclaw consumer.** Subagent roles are inline-injected (L5 B-opt2), state is
   session-scoped in `.codexclaw/`, and there is no child-completion ledger that must flush on
   SubagentStop in the MVP. (Phase 2 L24 subagent-config store is file-based + read on spawn, not on
   SubagentStop.)

## Revisit triggers (when this decision should be reopened)
- If R-8 research finds UserPromptSubmit does NOT fire on native-goal autonomous continuation turns,
  a PostCompact (or other) injection surface becomes necessary -> reopen.
- If a future loop adds a child-completion ledger that must be flushed deterministically -> SubagentStop.
