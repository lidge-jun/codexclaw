## Goal (one slice)
Freeze the interview result as a plan artifact that later goal-mode PABCD consumes read-only.

## Resolved (jun 2026-06-30) -- 핸드오프 타깃 = codex 네이티브 goal
freeze는 **plan artifact(파일+manifest+hash)** 를 굳히는 것이고, 그 다음 "goal 시작"은 codex 내장
`create_goal`을 호출해 **네이티브 goal**을 생성하는 것이다. codexclaw는 자체 goal-start FSM이나 goal
저장소를 만들지 않는다. frozen plan은 그 네이티브 goal이 read-only로 소비하는 spec 표면일 뿐이다.
즉: interview ready → freeze(plan manifest) → `create_goal(objective)`(codex가 소유·자율 continuation
구동) → goal 모드에서 codexclaw FSM이 PABCD를 돌린다. 근거: `codex-rs/core/src/tools/handlers/goal.rs:23`
(create_goal), `codex-rs/core/src/goals.rs:1271`(자율 continuation).

## 보강 (007 findings R-5/R-7) -- evidence bundle + 활성화 bridge
- **R-5 evidence carry**: freeze manifest는 objective+hash만이 아니라 **구조적 evidence bundle**을 담는다 —
  dimensions, OPEN ASSUMPTIONS, contradictions[], seed/acceptance-criteria, research report. goal 진입 시 이
  bundle ref를 주입해 PABCD가 소비한다(cli-jaw seedSpec/researchReport parity). objective+hash만으로 핸드오프 금지.
- **R-7 활성화 bridge**: plugin은 `create_goal`을 직접 호출 못 한다(fork 아님). 따라서 freeze 후 **활성화
  directive**를 둔다: "①`get_goal`로 현재 스냅샷 확인 → ②objective-only `create_goal` 호출"을 skill/hook 텍스트로
  지시하고, 이후 `goals_1.sqlite`에 goal row가 실제 생성됐는지 **검증**한다(omo codex-goal-instruction.ts:51 parity).
  생성 실패 시 goal 모드 진입 안 됨을 명확히 보고.

## Scope (decision-complete)
- Files to add/edit: `plugins/codexclaw/components/pabcd-state/src/cli.ts`, `plugins/codexclaw/components/pabcd-state/src/state.ts`, `plugins/codexclaw/components/pabcd-state/test/cli.test.ts`, `plugins/codexclaw/components/pabcd-state/test/state.test.ts`, and `.codexclaw/plan/` format docs if fixtures are owned here.
- Canonical spec surface is under `.codexclaw/plan/`.
- Freeze records the current plan revision with a manifest containing file list, sha256, and frozen timestamp.
- The frozen plan must include `## OPEN ASSUMPTIONS`; its content is part of the hash.
- Goal start recalculates the current plan hash and compares it to the manifest.
- If the file changed, warn that the canonical plan changed, re-freeze the current file, and refuse stale manifest execution.
- `/goal` without prior interview remains allowed; lightweight goal backfill is autonomous verification, not Interview.
- Handoff target is codex native `create_goal` (objective-only, per L3 budget gate); the frozen plan is the read-only spec the native goal consumes.
- Must-NOT-Have: no immutable seed YAML clone, no stale execution, no goal-mode user questions, no devlog-only requirement, NO self-built goal-start FSM / goal store (codex owns goal state).

## Hardening pins (Pass 1, jun 2026-06-30) -- exact manifest
- Freeze manifest path is EXACT: `.codexclaw/interview/freeze.json` (per-session under the codexclaw
  state dir). The canonical plan content lives under `.codexclaw/plan/`.
- Manifest schema: `{ frozenAt: ISO8601, planFiles: [{path, sha256}], planHash: sha256(concat of
  per-file sha256 in path order), objective: string, slug: string, evidenceBundle: {dimensions,
  openAssumptions[], contradictions[], acceptanceCriteria[], researchReportRef} }`.
- `slug` derivation: lowercase objective, non-alphanumeric -> `-`, collapse repeats, trim, cap 48
  chars (parity with existing `state.ts` slug rule).
- Goal-start integration: the activation directive (R-7) recomputes `planHash` and compares to
  `freeze.json`; mismatch -> refuse stale execution + re-freeze. codexclaw never calls `create_goal`
  itself (R-7); it instructs the runtime to call objective-only `create_goal`.
