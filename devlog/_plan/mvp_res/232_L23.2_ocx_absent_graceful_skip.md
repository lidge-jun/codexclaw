# L23.2 (Decade 232) -- ocx Absent Graceful Skip

Status: RESOLVED
Cluster: 4 - Phase: 2 - Shorthand: cxc
Source-of-record: 230_L23_provider_bridge.md, 260629_codexclaw_mvp/031_provider_bridge.md, 030_phase2_overview.md, 000_research.md

## Goal (one slice)
Make ocx absence a clean native-catalog state: SessionStart exits 0, codexclaw
loads, and downstream catalog/GUI surfaces know that Codex-native models remain
available.

## Why now / dependencies
- Upstream: depends on L23.1 detection helpers and on D2 from research:
  ocx-free users still get dev skills, IPABCD, and default-model subagents.
- Downstream: prevents L25, L27, and L28 from treating missing ocx as a failed
  plugin start.

## Scope (decision-complete)
- Files to add/edit:
  - `plugins/codexclaw/components/provider-bridge/src/cli.ts`
  - `plugins/codexclaw/components/provider-bridge/test/cli.test.ts`
  - `plugins/codexclaw/hooks/session-start-ensuring-provider-bridge.json`
- Exact behavior:
  - If no `ocx` command and no known service are found, return exit 0.
  - Emit provider status such as native-catalog / ocx-absent for L25 and L27.
  - Keep diagnostics clear enough for `cxc doctor` wording later.
  - Do not hide a real detected-ocx read failure behind the absent path.
- Must-NOT-Have:
  - No session failure merely because opencodex is not installed.
  - No fabricated non-ocx model source beyond Codex's native catalog.
  - No GUI link to `http://localhost:10100` when ocx is absent.

## IPABCD micro-cycle
- I: not interview-bearing; Q-P2-2/Q3 resolved absent ocx as Codex-native
  catalog, not single-model.
- P: add absent-state status, skip branch tests, and diagnostics that distinguish
  absent from present-but-failed.
- A: audit angle = "does graceful skip preserve correct failure semantics?"
  Reviewer checks no-op success, detected-read failure, and service-probe
  failure.
- B: implement absent branch, normalized status output, and fixture tests with
  PATH stripped.
- C: run node:test for absent PATH; run provider bridge directly and inspect exit
  code plus stdout.
- D: done = S6 absent path passes and codexclaw remains usable with
  native-catalog subagents.

## Acceptance (1-3 testable criteria)
1. With no ocx on PATH and no service, provider bridge exits 0.
2. Status output says native-catalog / ocx-absent and contains no fabricated
   provider claims.
3. Present-but-failing ocx list read is still a reported failure.
