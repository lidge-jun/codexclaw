## Goal (one slice)
Define and implement the scheduled-run result destination for jobs launched by
L29, with project-local file storage recommended and channel delivery deferred.

## Why now / dependencies
- Upstream: depends on L29 job store and generated scheduler command contract.
- Downstream: gives L30.2 evidence to inspect and gives L31 a stable result
  source to forward later.

## Scope (decision-complete except Q-P3-2)
- Files to add/edit:
  - result writer/reader under the scheduler CLI surface
  - project-local result paths under `.codexclaw/`
  - tests for success, failure, retention, and redaction if applicable
- Exact behavior:
  - Capture prompt metadata, start/end timestamps, exit status, stdout/stderr
    summary, and result body or body location.
  - Expose `cxc schedule runs [job-id]`.
  - Expose `cxc schedule show-run <run-id>`.
  - Preserve non-zero `codex exec` failures for later inspection.
- Must-NOT-Have:
  - No telegram/discord integration.
  - No server-owned bgtask model.
  - No silent discard of stdout or stderr.
  - No fabricated success when `codex exec` exits non-zero.

## IPABCD micro-cycle
- I: not interview-bearing; Q-P3-2 decides result destination.
- P: define run-result schema, storage path, reader commands, capture wrapper,
  and failure handling.
- A: audit angle = "can a user inspect scheduled-job failure after the fact?"
  Reviewer checks non-zero exit capture and result readability.
- B: implement writer, list/show commands, stdout/stderr summarization, and
  fixture tests.
- C: run node:test for run results; run a harmless generated command once and
  inspect recorded output.
- D: done = scheduled results are inspectable without messaging channels.

## Acceptance (1-3 testable criteria)
1. A run record includes prompt metadata, timestamps, exit code, stdout/stderr
   summary, and result body or location.
2. `cxc schedule runs` and `show-run` expose success and failure records.
3. Channel delivery is not required to inspect the result.

## QA channel (node:test path / CLI stdout / tmux / data dump)

- CLI stdout: `cxc schedule runs` and `cxc schedule show-run <run-id>`.
- Data dump: seeded run result under `.codexclaw/`.
