## Goal (one slice)
Close two foundational FSM principle gaps the 007 cross-reference (cli-jaw/jawcode/omo) found in the
shipped L1 engine, by editing real code in `pabcd-state`:
- R-4: add an explicit **IDLE** closed/rest phase so a cycle actually closes (`D -> IDLE`) before the
  next work-phase `P` can open. Shipped code had `phase` default `I` and `nextPhase(D)=null` (never
  returns to a rest state), contradicting the "D returns to IDLE to bound scope drift" design.
- R-2: make the audit/check evidence gate **structural**, not decorative. Shipped `fsm.ts` exposes
  `auditPassed`/`checkPassed` flags that NO code path ever sets, and `A`/`C` entry is unconditionally
  open, so the evidence gate survived only as prompt prose. Port cli-jaw's `GATED_TRANSITIONS` +
  attestation contract: `A->B` and `C->D` require a valid attestation object, and the gate flags flip
  to true ONLY inside `transition()` when that attestation validates.

## Why now / dependencies
L8+ (interview freeze -> goal handoff) trusts the FSM to actually close cycles and to gate forward
motion on evidence. If `D` never closes to IDLE and the gate flags are unsettable, every later loop
inherits a decorative state machine. This must land before Cluster 1 build work.

  - `transition(state, to, attest?): { ok; state?; reason? }`
- Gate semantics (port from cli-jaw):
  - `A->B` is gated: requires an attestation with non-empty `did`. On success `flags.auditPassed=true`.
  - `C->D` is gated: requires `did` + non-empty `checkOutput` + `exitCode===0`. On success
    `flags.checkPassed=true`.
  - Forward gated transition WITHOUT a valid attest is rejected (`ok:false`); no flag flips, no
    phase change (fail-closed, cli-jaw 409 parity).
  - `D->IDLE` closes the cycle: phase=IDLE, `orchestrationActive=false`, gate flags reset.
  - `IDLE->P` is valid without an interview flag (interview is optional); `I->P` still requires
    `flags.interview` (preserves L1.2 contract).
- Must-NOT-Have:
  - No change to hook stdin/stdout envelope shape.
  - No server dependency (`transition` is pure; callers persist via `writeState`).
  - No auto-advance from passive `Stop`.
