## Goal (one slice)
Define the main-session question generator used to turn high-severity contradictions into user choices.

## Why now / dependencies
L8 readiness and L9 normalized contradictions must exist. This sub-pass keeps all user interaction in the main session.

## Scope (decision-complete)
- Files to add/edit: `plugins/codexclaw/components/pabcd-state/src/hook.ts`, `plugins/codexclaw/components/pabcd-state/test/hook.test.ts`.
- Every question includes background, why the decision matters, and where the answer changes the plan.
- Every question is concrete; avoid abstract labels that do not explain operational meaning.
- Provide two or three options.
- Put the recommendation first.
- Include one impact or tradeoff sentence per option.
- Use `request_user_input`; assistant-emitted choice fences are not the primary selector.
- Subagents never generate or deliver questions.
- Pending-question behavior is main-session discipline unless a later state API is chosen; unrelated free-form answers should be refused or restated.
- Must-NOT-Have: no plain-text fallback for missing `request_user_input`, no hidden assumptions outside `## OPEN ASSUMPTIONS`.

## IPABCD micro-cycle
- I: Main receives high-severity contradictions and asks exactly the next useful choice.
- P: Add reusable directive text for question shape and option ordering.
- A: Evaluator checks that each generated question is answerable and not leading without disclosure.
- B: Implement hook directive text and tests for background, recommendation-first options, and impact text.
- C: Run `node --test plugins/codexclaw/components/pabcd-state/test/hook.test.ts`.
- D: Done = high-severity contradictions can be converted into `request_user_input` prompts without worker involvement.

## Acceptance (1-3 testable criteria)
1. Question rules mention background, concrete options, recommendation first, and impact text.
2. Directive keeps user prompting in the main session only.
3. Directive references `request_user_input` as the interaction path.

- Manual dry-run: generated prompt text can be read as a 2-3 option selector.
