## Goal (one slice)
Make interview state bounded, fail-closed, and idempotent enough for repeated interview rounds.

## Why now / dependencies
L8.1 and L8.2 create state and readiness. This sub-pass closes T2, T3, and T6 before L9/L10 add repeated contradiction and question cycles.

## Scope (decision-complete)
- Files to add/edit: `plugins/codexclaw/components/pabcd-state/src/state.ts`, `plugins/codexclaw/components/pabcd-state/test/state.test.ts`.
- T2: cap in-state arrays for assumptions, contradictions, dimension known, and dimension unknown; keep the hot session JSON small.
- If detailed history is required later, externalize it to an append-only ledger outside this sub-pass; do not grow `State` unbounded.
- T3: corrupted tracker data must fail closed for readiness. Dropping malformed entries must not accidentally produce ready state.
- Reconstruct invalid dimension levels to low and invalid confidence to 0.
- Reconstruct invalid or legacy assumptions with `recorded:false` so they require explicit re-recording before readiness.
- Reconstruct invalid contradictions conservatively: valid ones stay, invalid tracker shape forces not-ready through low/default dimensions.
- T6: do not reuse only the existing 50-turn injection cap as the idempotency horizon.
- Add operation-level identifiers in tracker-adjacent data if implementation needs replay defense: round id, contradiction correlation id, plan edit id, or freeze id.
- Must-NOT-Have: no silent ready state after lossy reconstruct, no unbounded evidence blobs, no hidden ledger outside documented paths.

## IPABCD micro-cycle
- I: Before each round writes state, trim arrays and preserve enough summary to ask or freeze safely.
- P: Define caps as constants in `state.ts` with tests proving truncation behavior.
- A: Reviewer checks that malformed persisted JSON cannot satisfy `isInterviewReady()`.
- B: Implement bounded reconstruct helpers and fail-closed tests.
- C: Run `node --test plugins/codexclaw/components/pabcd-state/test/state.test.ts`.
- D: Done = state stays bounded and corrupted interview data cannot advance to P.

## Acceptance (1-3 testable criteria)
1. Oversized arrays are capped deterministically during reconstruct or write normalization.
2. Malformed tracker data reconstructs to not-ready state.
3. Legacy assumptions without `recorded:true` block readiness until recorded.

## Hardening pins (Pass 1, jun 2026-06-30) -- exact caps + IDs
- **Numeric caps (T2)**: `MAX_TRACKER_ARRAY = 50` per array (`assumptions`, `contradictions`, and
  each `DimensionScore.known` / `.unknown`). Truncation direction = DROP-OLDEST (keep most recent 50);
  define as an exported const in `state.ts` and assert in `state.test.ts`.
- **Replay/operation IDs (T6)**: name the fields explicitly -- `roundId: number` (monotonic per
  interview), `contradictionId: string` (correlates a Mind finding, see L9.2 `correlationId`),
  `planEditId: string`, `freezeId: string`. These live on tracker-adjacent records, not as unbounded
  blobs.
- **Ledger scope (resolves L8<->L10 ambiguity; disambiguated from the L1 audit ledger)**: Cluster 1
  adds NO *new interview-history ledger* and NO `ledger_only` closure surface. The interview tracker's
  durable surface is bounded in-state arrays only. This does NOT remove the existing L1 phase-audit
  ledger (`.codexclaw/ledger.jsonl`, append-only, written by the FSM on phase transitions per
  010_L1) nor the L10.3 freeze manifest (`.codexclaw/interview/freeze.json`) — those remain. Three
  distinct surfaces: (a) L1 FSM audit ledger = phase-transition provenance; (b) bounded in-state
  interview arrays = tracker working set; (c) L10.3 freeze manifest = handoff spec. L10 closure uses
  `assumptions_only` (recorded assumptions), never a hidden interview-history ledger.
