# L9.2 (Decade 092) -- Contradiction-Only Dispatch Protocol

Status: PLANNED
Cluster: 1 - IPABCD Interview completion - Phase: 1 - Shorthand: cxc
Source-of-record: 080.1, 080.2, 034.5, ouroboros 030/040

## Goal (one slice)
Specify the dispatch envelope and validation rules that keep Mind workers read-only contradiction producers.

## Why now / dependencies
L9.1 defines Mind prompts. T4/T7 are resolved, so the main agent dispatches from the top-level session and validates returned contradictions.

## Scope (decision-complete)
- Files to add/edit after unblock: `plugins/codexclaw/components/pabcd-state/src/hook.ts`, `plugins/codexclaw/components/pabcd-state/src/state.ts`, `plugins/codexclaw/components/pabcd-state/test/hook.test.ts`, `plugins/codexclaw/components/pabcd-state/test/state.test.ts`.
- Dispatch input includes current plan/spec context, relevant state summary, selected Mind id, and a correlation key.
- Dispatch output is a JSON array of contradictions only.
- Each item must have `dimension: goal | constraint | success | ontology`, `contradiction`, `severity: low | medium | high`, and `evidence`.
- `evidence` must be a file:line, section reference, or exact short source quote; unsupported guesses are rejected.
- Empty array means that Mind found no contradiction.
- Malformed Mind output is ignored or rejected and must not mark interview ready.
- Subagents never ask questions, edit files, call the user, select options, write state, or update plans.
- Must-NOT-Have: no natural-language answer paragraphs as accepted output, no side effects from subagents, no hidden state writes.

## IPABCD micro-cycle
- I: Top-level main selects Minds and sends context; worker returns contradictions only.
- P: Add protocol text and normalization helpers after boundary decision.
- A: Reviewer tries to smuggle a question or edit instruction through the output and expects rejection.
- B: Implement validation and correlation preservation.
- C: Run hook/state tests with valid, empty, and malformed outputs.
- D: Done = accepted contradictions are bounded, evidence-bearing, and correlated to a Mind.

## Acceptance (1-3 testable criteria)
1. Directive forbids subagent questions, edits, user calls, and state writes.
2. Validation rejects missing dimension, invalid severity, or missing evidence.
3. Every accepted contradiction carries the Mind correlation key.
