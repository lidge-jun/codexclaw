## Conflict classes

### C1 — Frontmatter schema mismatch
- cli-jaw uses `metadata: { "keywords": [...] }`. codex-rs `SkillFrontmatter` does NOT read
  `keywords`; only `name`, `description`, `metadata.short-description`. `deny_unknown_fields`
  means stray keys can ERROR.
- ACTION: rewrite frontmatter to the exact codex schema; move intent into `description`.

### C2 — "Injected by orchestrator" assumption
- e.g. `dev/SKILL.md:3` "...Always injected by orchestrator." cli-jaw injects skills via the
  boss/orchestrator. codex AUTO-ROUTES via `description` + `policy.allow_implicit_invocation`.
- ACTION: remove "injected by orchestrator" framing; rewrite description as trigger phrases;
  set allow_implicit_invocation where always-on behavior is wanted.

### C3 — `cli-jaw orchestrate I/P/A/B/C/D` commands
- dev-pabcd is built around literal `cli-jaw orchestrate ...` (dev-pabcd:17-61). These commands
  do NOT exist in codex; they target the cli-jaw server.
- ACTION: replace with codexclaw-native IPABCD (file FSM + hooks, steps 022/022.1). IPABCD becomes
  a guide + hook-driven flow, not a server command. Reconcile dev-pabcd INTO codexclaw `pabcd`.

### C4 — boss/employee / dispatch model
- dev/SKILL.md references boss, employees, `cli-jaw dispatch`, "the server re-invokes the boss".
  codex has no boss/employee server; it has runtime subagents (`multi_agent_v1.*`, step 025).
- ACTION: remap "employee" → codex subagent role (.toml). Remove server re-invoke / bgtask
  wording. "boss" → the main agent.

### C5 — devlog / structure / verify-counts house rules
- dev & dev-scaffolding mandate cli-jaw repo artifacts (`structure/`, `devlog/`, decade numbering,
  "MUST NOT create devlog/AGENTS.md silently"). These are cli-jaw-PROJECT rules, not universal.
- ACTION: keep the UNIVERSAL discipline (read source-of-truth docs first; don't scaffold silently)
  but drop cli-jaw-specific paths/filenames. Decade numbering is optional guidance, not a hard rule.

### C6 — task_tags / overlay / class system tied to dispatch
- §0 work classifier (C0-C5) references `task_tags` set "by the boss at dispatch" and overlay
  activation. The CLASSIFIER ITSELF is valuable and universal; the dispatch plumbing is not.
- ACTION: keep the classifier as self-assessment guidance; strip dispatch/task_tags plumbing.

### C7 — goal mode wording
- "you are the boss/goal agent inside cli-jaw, do not block the turn polling it" + bgtask/web-ai.
- ACTION: replace with codex goal model (native create_goal + omo-style gate, step 023). Remove
  bgtask/session-poll wording (cli-jaw runtime specific).

### C8 — Cross-skill references by cli-jaw path
- Skills point to sibling skills as `dev-pabcd/SKILL.md` etc., and to `/Users/jun/.cli-jaw-3459/...`.
- ACTION: rewrite cross-references to codexclaw skill names; never hardcode the cli-jaw home path.
