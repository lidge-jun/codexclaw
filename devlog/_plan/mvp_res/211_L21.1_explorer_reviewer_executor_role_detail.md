## Goal (one slice)
Define the three MVP inline subagent roles in operational detail:
`explorer`, `reviewer`, and `executor`. These roles are prompt templates injected
into `spawn_agent({message})`, not installed Codex role files in MVP.

## Why now / dependencies
- Upstream: L5 chose Pass-5 B-opt2 inline roles; L20.2 defines spawn/wait polling.
- J-8 keeps role registration inline-only until plugin role pickup is verified.
- J-9 accepts prompt-only read-only discipline; no tool allowlist enforcement is
  added in MVP.
- Downstream: Phase 2 can promote the same taxonomy into first-class configurable
  roles once L24/L27 exist.

## Scope (decision-complete)
Files to add/edit in implementation:
- `plugins/codexclaw/roles/explorer.toml` or source-of-truth role template path
- `plugins/codexclaw/roles/reviewer.toml`
- `plugins/codexclaw/roles/executor.toml`
- Inline role injection helper used by subagent orchestration
- Role template tests

Exact behavior:
- `explorer` is read-only. It searches files, docs, history, or external sources
  when explicitly in external variant, then returns concise findings with paths,
  commands, evidence, and open questions. It does not edit.
- `reviewer` is read-only. It audits plans, diffs, risks, acceptance, and evidence;
  it returns findings with severity and blockers. It does not edit.
- `executor` may edit within the assigned scope. It performs the smallest correct
  change, records evidence, and avoids unrelated fixes.
- Every role invocation includes the project root, task, scope boundaries, allowed
  surfaces, expected output shape, and a reminder to stop on ambiguity.
- Roles share one namespace with future `mind-*` roles by prefix convention, but
  MVP role calls remain inline prompt variants.

Must-NOT-Have:
- No config `[agents]` install or plugin manifest agents field in MVP.
- No claim that read-only roles are technically sandboxed.
- No subagent allowed to continue after the main turn ends.
- No role proliferation beyond the three functional roles plus documented variants.

2. Explorer and reviewer templates explicitly say no edits/findings only.
3. Executor template permits edits only inside assigned scope.
