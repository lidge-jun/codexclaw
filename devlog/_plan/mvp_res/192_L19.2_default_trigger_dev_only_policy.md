# L19.2 (Decade 192) -- Default Trigger Dev-Only Policy

Status: BLOCKED(L12-L17)
Cluster: 2 (part B) · Phase: expansion · Shorthand: cxc
Source-of-record: 190_L19_skill_hub_rewrite.md; 260629_codexclaw_mvp/100_skill_hub.md; 090.1 J-4/J-6

## Goal (one slice)
Lock the default implicit skill set to the resolved policy: `dev` only.
`search`, host-provided `pdf`, `skill-hub`, all role-specific `dev-*` routers,
and artifact skills remain on-demand with explicit mention or `dev` hub routing
required.

## Why now / dependencies
- Upstream: L12-L17 determine the final `dev-*` router names and confirm `pabcd`
  is the explicit/hook-driven workflow skill.
- J-4/J-6 resolved the earlier conflict between broad dev-skill auto-triggering
  and a small default prompt. The default discipline is `dev`; role-specific
  skills are loadable but not always-on.
- Downstream: this policy is the prompt-size and trigger-noise contract for all
  later skills such as diagram, xlsx, pptx, video, and code-intelligence add-ons.

## Scope (decision-complete)
Files to add/edit after unblock:
- `plugins/codexclaw/skills/dev/agents/openai.yaml`
- `plugins/codexclaw/skills/search/agents/openai.yaml` (`allow_implicit_invocation: false`)
- No codexclaw PDF policy file; PDF is host-provided and remains non-implicit for codexclaw policy
- `plugins/codexclaw/skills/skill-hub/agents/openai.yaml` (`allow_implicit_invocation: false`)
- `plugins/codexclaw/skills/*/agents/openai.yaml` for all on-demand skills
- `plugins/codexclaw/skills/skill-hub/references/catalog.md`

Exact behavior:
- Default implicit-visible set is exactly `{dev}`.
- `dev` carries always-on development discipline and routes agents toward the
  correct role-specific skill when needed.
- `search` is on-demand; current/web lookup still uses explicit triggers and its
  Korean intent guard when loaded.
- `pdf` is host-provided by Codex, not a codexclaw skill, and is not added to
  the codexclaw implicit set.
- `skill-hub` is on-demand as the bootstrap pointer for hidden capabilities,
  reached by explicit trigger or `dev` hub routing.
- All `dev-architecture`, `dev-debugging`, `dev-backend`, `dev-data`,
  `dev-frontend`, `dev-uiux-design`, `dev-testing`, `dev-code-reviewer`,
  `dev-security`, `dev-devops`, `dev-scaffolding`, and `pabcd` policy decisions
  must be reconciled with L12-L17 final outputs before writing.

Must-NOT-Have:
- No broad implicit exposure of all `dev-*` routers.
- No default implicit artifact skills, including host-provided `pdf`.
- No hidden skill disabled as a substitute for implicit-off.
- No policy drift between `catalog.md` and `agents/openai.yaml`.

## IPABCD micro-cycle
- I: not interview-bearing.
- P: build a policy table from actual skill dirs, mark only `dev` implicit, mark
  all others on-demand, then generate/update `openai.yaml` files.
- A: reviewer compares rendered prompt to the policy table and confirms the set is
  exactly one name.
- B: edit policy files and catalog classification in one pass.
- C: `codex debug prompt-input` plus node:test policy-table assertion.
- D: done = default prompt is small and all specialized skills stay reachable by
  explicit mention.

## Acceptance (1-3 testable criteria)
1. Rendered implicit skills equal exactly `dev`.
2. Every other skill has `allow_implicit_invocation: false`, not disabled.
3. Catalog labels match policy files for every skill.
