FEM의 Handoff 판단 구조

🧠 FEM이 판단한다 핵심

FEM은 시스템 프롬프트로 "코딩 작업은 위임하라"는 지시를 받음. 사용자 발화를 듣고 FEM 스스로 handoff 여부를 결정.

직접 응답 (No Handoff)
"몇 시야?" / "리액트가 뭐야?"
일반 대화, 지식 질문
→ FEM 바로 음성 답변
→ Orchestrator 호출 없음
Handoff (위임)
"이 버그 고쳐줘" / "테스트 돌려봐"
코드 실행, 파일 수정 필요
→ handoff request 전송
→ Orchestrator 실행 후 결과 반환
⚠️ 판단 오류 보완 — 모든 realtime 턴에 voice-ready final response 강제 (#27114)
Handoff 시 전달되는 데이터

  <input>이 함수 리팩터링 해줘</input>

</realtime_delegation>

transcript_delta
